## ADDED Requirements

### Requirement: Offline Fusion Weight Ablation Harness

The project SHALL provide an offline research tool at `scripts/ablation_fusion_weights.py` that scores candidate 5-signal fusion weight sets against a learning-event corpus and reports logloss, AUC, and expected calibration error for each candidate.

The harness SHALL mirror the production fusion rule (`backend/app/services/mastery_fusion.py::compute_fused_mastery`: dynamic weight renormalization over active signals, clamp to `[0, 1]`) using a plain-Python equivalent so the harness has zero backend imports and runs without `pip install`.

The harness SHALL be invocable from any developer worktree using only the Python 3.10+ standard library. It SHALL NOT be wired into CI, lefthook, or any pre-commit / pre-push / pre-merge gate (per user decision 2026-04-07 Q4).

The harness SHALL accept either a `--synthetic N` flag (generate N rows from an in-memory BKT + FSRS-driven simulator) or an `--input PATH` flag (load rows from a JSONL file whose row schema is `{node_id, y, signals: {bkt, fsrs, exam, cal, conf}}`). Both input modes SHALL produce the same report format so real-data runs and synthetic-data runs are directly comparable.

The harness SHALL include a `sanity_check_math()` function that verifies the metric formulas against a perfect-predictor edge case, aborting with a non-zero exit code if the check fails.

#### Scenario: Synthetic run produces a report with all candidates

- **GIVEN** a developer runs `python scripts/ablation_fusion_weights.py --synthetic 1000 --seed 42 --output /tmp/report.json`
- **WHEN** the script completes successfully
- **THEN** `/tmp/report.json` exists and parses as JSON
- **AND** the report's `candidates` field contains all 6 keys: `baseline`, `no_exam`, `no_cal_conf`, `bkt_only`, `fsrs_only`, `equal`
- **AND** each candidate's `logloss`, `auc`, `ece` values are finite floats (no NaN)
- **AND** the report's `data_source` field identifies the run as synthetic

#### Scenario: Fixture run matches the script's own synthetic distribution

- **GIVEN** the committed fixture at `scripts/fixtures/synthetic_fusion_sample.jsonl` containing 30 rows generated by the script's own `generate_synthetic(30, seed=42)`
- **WHEN** the developer runs `python scripts/ablation_fusion_weights.py --input scripts/fixtures/synthetic_fusion_sample.jsonl`
- **THEN** the script completes with exit code 0
- **AND** the printed summary table lists all 6 candidates
- **AND** the candidate rankings are consistent with the math (fsrs_only has lowest AUC; no candidate has NaN metrics)

#### Scenario: Sanity check catches metric drift

- **GIVEN** a hypothetical future edit that breaks one of the metric functions (`logloss`, `auc`, or `calibration_error`)
- **WHEN** the developer runs `python scripts/ablation_fusion_weights.py --synthetic 100`
- **THEN** `sanity_check_math()` rejects the edit
- **AND** the script prints `[FAIL] sanity_check_math() rejected the metric impls — aborting.`
- **AND** exits with a non-zero status (code 2)

#### Scenario: Harness is not wired into CI

- **GIVEN** the current `lefthook.yml` configuration
- **WHEN** a developer runs `git commit`, `git push`, or any other git hook-triggered event
- **THEN** `scripts/ablation_fusion_weights.py` is NOT invoked by any hook
- **AND** the pre-push A11 regression suite makes no reference to the harness
- **AND** removing or modifying the harness does not cause any automated gate to fail

#### Scenario: Harness has zero backend imports

- **GIVEN** the current state of `scripts/ablation_fusion_weights.py`
- **WHEN** a reviewer runs `grep -E "^from backend|^import backend|from app\\." scripts/ablation_fusion_weights.py`
- **THEN** the search returns no matches
- **AND** the script's import block contains only Python standard library modules (`argparse`, `dataclasses`, `json`, `math`, `pathlib`, `random`, `statistics`, `typing`)
