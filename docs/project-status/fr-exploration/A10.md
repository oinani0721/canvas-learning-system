FSRS 在检验白板中的实际作用：    
                                  
    你在检验白板回答了"万有引力"的问题
         │
         ▼
    系统给你打了 75 分
         │
         ├──▶ 存入 Neo4j: Episode ─[:SCORED {score:75}]─▶ Node
         │
         └──▶ 更新 FSRS 卡片状态：
              stability: 3.2 → 4.1（记忆更稳定了）
              difficulty: 0.6（中等难度）
              state: Review（复习阶段）
              下次复习: 4天后

    ═══════ 下次你再检验这个概念时 ═══════
         │
         ▼
    系统查 FSRS 历史：[65, 72, 78, 70, 75]
         │
         ├── 平均分 72，趋势"稳定"
         │
         ▼
    影响出题难度：

    ┌─────────────────────────────────────────────┐
    │  平均分 < 60  → 出基础题（定义、概念）      │
    │  平均分 60-79 → 出验证题（理解深度）        │
    │  平均分 ≥ 80  → 出应用题（跨概念、实际场景）│
    └─────────────────────────────────────────────┘

    还会把你的得分趋势告诉 AI：
    "这个学生最近5次得分是 [65,72,78,70,75]，趋势稳定"
    → AI 据此调整提问方式和评分标准

  FSRS 变量在代码中的调用链：

    用户回答 → verification_service.process_answer()
         │
         ▼
    AI 评分 → score = 75（0-100）
         │
         ▼
    映射为 FSRS 等级：
    0-39 → Grade 1 (Forgot)
    40-59 → Grade 2 (Struggled)
    60-79 → Grade 3 (Correct)    ← 75分走这里
    80-100 → Grade 4 (Fluent)
         │
         ▼
    mastery_engine.update_on_interaction(concept, grade=3)
         │
         ├── BKT 更新 p_mastery（掌握概率）
         │
         └── fsrs_manager.review_card(card, Rating.Good)
              │
              ▼
         更新 FSRS 卡片：
         - fsrs_stability（记忆稳定性）
         - fsrs_difficulty（概念难度）
         - fsrs_state（学习阶段）
         - fsrs_reps（复习次数+1）
         - fsrs_lapses（遗忘次数）
```

  关键：FSRS 数据和关系图谱是分开存储的，不在同一层：

  ┌────────────────────────┬───────────────────────┬────────────────┐
  │          数据          │        存在哪         │      用途      │
  ├────────────────────────┼───────────────────────┼────────────────┤
  │ 概念关系 (RELATES_TO)  │ Neo4j 图谱            │ RAG 检索上下文 │
  ├────────────────────────┼───────────────────────┼────────────────┤
  │ 白板连线 (CANVAS_EDGE) │ Neo4j 图谱            │ 推荐系统       │
  ├────────────────────────┼───────────────────────┼────────────────┤
  │ FSRS 卡片状态          │ ConceptState 内存对象 │ 决定复习间隔   │
  ├────────────────────────┼───────────────────────┼────────────────┤
  │ 历史评分               │ Neo4j SCORED 关系     │ 调整出题难度   │
  └────────────────────────┴───────────────────────┴────────────────┘

  ★ Insight ─────────────────────────────────────
  这个系统实际上有四层独立的数据模型，各自解决不同问题：图谱管"关系"、向量管"相似度"、FSRS
  管"时间"、评分管"难度"。这不是设计缺陷——成熟的学习系统（如
  Anki、Duolingo）也是多模型并行的。但问题是这四层之间的信息流是否畅通。目前发现 RAG 字段名不匹配和 CONNECTS_TO
  死代码两个断裂点，说明各层之间的胶水代码需要修复。
