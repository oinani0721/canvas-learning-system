**版本**: v1.0
**最后更新**: 2025-10-15
**目标用户**: Canvas学习系统用户

## 📖 概述

### 什么是检验白板？

检验白板是Canvas学习系统的核心创新功能，它是一个**动态学习画布**，用于验证和深化你对知识的理解。

**核心理念**：
> 检验白板不是静态的测试卷，而是一个可以持续生长的知识网络。通过迭代学习循环，你可以将检验白板从简单的问题列表，扩展成一个接近原白板复杂度的完整知识体系。

### 检验白板的独特价值

| 特性 | 传统测试 | 检验白板 |
|------|---------|---------|
| **是否可扩展** | ❌ 固定题目 | ✅ 无限扩展 |
| **学习过程** | ❌ 一次性作答 | ✅ 迭代深化 |
| **反馈机制** | ❌ 仅给分数 | ✅ 评分+拆解+解释 |
| **知识网络** | ❌ 孤立问题 | ✅ 构建关联 |
| **复现原理** | ❌ 记忆答案 | ✅ 重建知识结构 |

### 检验白板能做什么？

在检验白板上，你可以使用**所有11个Sub-agent**进行操作，包括：

**拆解系列**（降低难度）：
- basic-decomposition - 基础拆解
- deep-decomposition - 深度拆解
- question-decomposition - 问题拆解

**解释系列**（补充理解）：
- oral-explanation - 口语化解释
- clarification-path - 澄清路径（最详细）
- comparison-table - 对比表
- memory-anchor - 记忆锚点
- four-level-explanation - 四层次解答
- example-teaching - 例题教学

**评分系统**（检验理解）：
- scoring-agent - 4维度评分

**Canvas操作**：
- canvas-orchestrator - 主控Agent

## 🔄 8步学习循环

检验白板的核心是一个**持续迭代的8步学习循环**：

### 步骤1：填写个人理解 📝

**操作**：在黄色理解节点中，用自己的话输出对检验问题的理解。

**要求**：
- 不看原白板材料
- 不查阅解释文档
- 凭记忆和理解输出
- 可以部分输出（不强制完整）

**示例**：
```
检验问题：什么是逆否命题？
黄色节点输出（你的理解）：
"逆否命题就是先对一个命题取反，然后再调换位置。
比如'如果下雨，那么地湿'的逆否命题是'如果地不湿，那么没下雨'。"
```

**Tips**：
- 即使不确定，也尝试输出你当前的理解
- 部分理解也有价值，评分系统会给出反馈

---

### 步骤2：发现不足 🔍

**操作**：回顾自己的输出，识别理解不足的地方。

**常见不足类型**：
1. **完全空白**：无法输出任何内容（红色节点）
2. **模糊不清**：能说出大概，但不准确（紫色节点可能性）
3. **缺少细节**：大框架对，但缺乏具体例子
4. **概念混淆**：与其他相似概念混淆

**自我诊断问题**：
- ❓ 我能准确定义这个概念吗？
- ❓ 我能举出具体例子吗？
- ❓ 我能区分易混淆概念吗？
- ❓ 我能解释背后的原理吗？

---

### 步骤3：继续拆解 🔧

**何时拆解**：
- 完全不懂（红色节点）
- 概念太复杂，无法直接理解
- 需要降低难度

**选择拆解Agent**：

| Agent | 使用场景 | 拆解结果 |
|-------|---------|---------|
| **basic-decomposition** | 概念复杂，需要分解 | 2-7个子问题 |
| **deep-decomposition** | 需要深层次理解 | 多层拆解 |
| **question-decomposition** | 需要检验型问题 | 针对性问题 |

**调用方式**：
```
"对节点 node-abc123 进行基础拆解"
或
"对'什么是逆否命题'这个问题进行拆解"
```

**拆解后的白板变化**：
- 增加2-7个子问题节点（红色）
- 增加对应数量的黄色理解节点
- 自动建立节点连接关系

---

### 步骤4：补充解释 📚

**何时补充解释**：
- 似懂非懂（紫色节点）
- 拆解后仍然不清楚
- 需要更深入的理解
- 需要记忆辅助

**选择解释Agent**：

| Agent | 适用场景 | 生成内容 |
|-------|---------|---------|
| **oral-explanation** | 需要通俗易懂的解释 | 800-1200字口语化解释 |
| **clarification-path** | 需要最详细的解释 | 1500+字深度解释 |
| **comparison-table** | 易混淆概念 | 结构化对比表格 |
| **memory-anchor** | 需要帮助记忆 | 类比、故事、口诀 |
| **four-level-explanation** | 需要渐进式理解 | 新手→专家4层解释 |
| **example-teaching** | 需要练习 | 完整例题+解答 |

**调用方式**：
```
"对节点 node-abc123 生成口语化解释"
或
"对比'逆否命题'和'否命题'的区别"
```

**解释后的白板变化**：
- 生成Markdown笔记文件（保存到与Canvas同目录）
- 在白板上创建file节点链接到笔记
- 可以点击file节点在Obsidian中查看笔记

---

### 步骤5：评分验证 ✅

**何时评分**：
- 填写完黄色节点后
- 想知道理解程度如何
- 决定是否需要进一步学习

**调用scoring-agent**：
```
"对节点 node-abc123 的黄色理解进行评分"
```

**评分维度**（4个维度，每个0-25分）：
1. **准确性**（Accuracy）：概念是否正确
2. **完整性**（Completeness）：是否覆盖关键点
3. **形象性**（Imagery）：是否有具体例子
4. **原创性**（Originality）：是否有个人理解

**评分结果示例**：
```json
{
  "总分": 75,
  "准确性": 20/25,
  "完整性": 18/25,
  "形象性": 20/25,
  "原创性": 17/25,
  "反馈": "定义基本正确，但缺少与'否命题'的区分说明。建议补充对比例子。",
  "建议行动": "调用comparison-table对比'逆否命题'和'否命题'"
}
```

---

### 步骤6：颜色流转 🎨

**颜色含义**：
- 🔴 **红色**（color: "1"）：不理解/未通过（评分<60）
- 🟣 **紫色**（color: "3"）：似懂非懂/待检验（评分60-79）
- 🟢 **绿色**（color: "2"）：完全理解/已通过（评分≥80）
- 🟡 **黄色**（color: "6"）：个人理解输出区

**颜色流转规则**：

```
初始状态：红色问题 + 空黄色理解
    ↓
[填写黄色节点]
    ↓
[调用评分]
    ↓
├─ 评分 ≥ 80 → 红色变绿色 ✅ (完全理解)
├─ 评分 60-79 → 红色变紫色 ⚠️ (部分理解)
└─ 评分 < 60 → 保持红色 ❌ (需继续学习)
    ↓
[紫色节点再次填写+评分]
    ↓
评分 ≥ 80 → 紫色变绿色 ✅
```

**颜色流转触发机制**：
- 评分系统自动根据分数更新节点颜色
- 你也可以手动在Obsidian中修改节点颜色（不推荐，会影响学习跟踪）

---

### 步骤7：添加自己的节点 ➕

**为什么添加节点**：
- 你有原创的理解和关联
- 你发现原白板遗漏的重要点
- 你想补充自己的笔记和心得

**可以添加的节点类型**：
1. **补充问题节点**：你认为重要但原白板未覆盖的问题
2. **原创理解节点**：你的个人见解和总结
3. **外部资源节点**：链接到有用的学习资源
4. **笔记节点**：你的学习心得和感悟

**如何添加**：
- 在Obsidian Canvas中直接创建节点
- 建议使用黄色或蓝色（避免与系统颜色冲突）
- 建立与相关节点的连接

**示例**：
```
原白板只讲了逆否命题的定义
你添加：
- "逆否命题在数学证明中的应用"（补充问题）
- "我的记忆口诀：先反后换"（原创理解）
- 链接到YouTube视频（外部资源）
```

---

### 步骤8：构建完整知识网络 🕸️

**目标**：通过多轮迭代，将检验白板扩展成接近原白板的知识网络。

**知识网络的标志**：
- ✅ 节点数量增长（从10个→30-40个）
- ✅ 层级结构形成（问题拆解成子问题）
- ✅ 解释文档丰富（多个file节点）
- ✅ 个人补充存在（你的原创节点）
- ✅ 绿色节点占比高（80%以上）

**知识网络示例**：

**初始检验白板**：
```
5个红色问题 + 5个黄色理解 = 10个节点
```

**第3轮迭代后**：
```
5个原始问题（3绿+2红）
+ 4个拆解子问题（2绿+2紫）
+ 3个解释file节点
+ 2个原创节点
= 14个节点（知识网络初具规模）
```

**最终状态**：
```
原始+拆解问题（20个节点，16绿+4紫）
+ 解释文档（8个file节点）
+ 原创补充（5个节点）
+ 关联连接（30+条边）
= 33个节点的完整知识网络
```

## 🎯 何时使用哪个Agent？决策树

### 决策树总览

```
开始：面对一个检验问题（红色）
    ↓
【判断1】我能输出任何内容吗？
    │
    ├─ NO（完全不懂）→ 调用 basic-decomposition
    │       ↓
    │   生成子问题 → 重新开始循环
    │
    └─ YES（能输出一些）→ 填写黄色节点 → 【判断2】
            ↓
【判断2】调用评分，分数多少？
    │
    ├─ ≥ 80分 → 节点变绿 ✅ → 继续下一个问题
    │
    ├─ 60-79分（紫色）→ 【判断3】
    │       ↓
    │   【判断3】缺少什么？
    │       ├─ 缺少深度理解 → clarification-path
    │       ├─ 易混淆概念 → comparison-table
    │       ├─ 需要记忆 → memory-anchor
    │       ├─ 需要例题 → example-teaching
    │       └─ 需要渐进式理解 → four-level-explanation
    │           ↓
    │       阅读解释 → 重新填写黄色节点 → 重新评分
    │
    └─ < 60分（红色）→ 【判断4】
            ↓
        【判断4】是概念问题还是难度问题？
            ├─ 难度太大 → basic-decomposition（拆解降难度）
            ├─ 需要通俗解释 → oral-explanation
            └─ 需要最详细解释 → clarification-path
                ↓
            学习后重新填写 → 重新评分
```

---

### 场景化决策指南

#### 场景1：完全不懂的概念（红色，无法输出）

**症状**：
- 黄色节点完全空白
- 不知道从何说起
- 看到问题就懵

**推荐流程**：
```
1. 调用 basic-decomposition
   → 将大概念拆解为2-7个小问题

2. 逐个击破子问题
   → 填写子问题的黄色节点

3. 如果子问题仍然不懂
   → 调用 oral-explanation 获得通俗解释

4. 学习解释后重新填写
   → 调用评分验证
```

**示例**：
```
问题："什么是群论？"（完全不懂）
    ↓
调用 basic-decomposition
    ↓
拆解为：
- "什么是群？"
- "群的四个运算法则是什么？"
- "群在数学中有什么作用？"
    ↓
逐个学习和输出
```

---

#### 场景2：似懂非懂的概念（紫色，60-79分）

**症状**：
- 能说出大概，但不准确
- 缺少细节和例子
- 与其他概念混淆

**推荐流程**：
```
1. 调用 scoring-agent
   → 看评分反馈指出的具体问题

2. 根据缺陷类型选择Agent：
   ├─ 缺少深度理解 → clarification-path
   ├─ 易混淆 → comparison-table
   ├─ 缺少例子 → example-teaching
   └─ 难以记忆 → memory-anchor

3. 阅读解释后重新输出
   → 调用评分，目标≥80分
```

**示例**：
```
问题："逆否命题和否命题的区别？"
初次输出：75分（紫色）
反馈："概念有部分理解，但未明确区分步骤"
    ↓
调用 comparison-table
    ↓
阅读对比表后重新输出
    ↓
再次评分：85分（绿色）✅
```

---

#### 场景3：需要深化理解（绿色，但想更深入）

**症状**：
- 已经达到80分（绿色）
- 想要专家级理解
- 想要应用到实际问题

**推荐流程**：
```
1. 调用 four-level-explanation
   → 获得新手→专家→创新的4层解释

2. 调用 example-teaching
   → 练习实际问题

3. 添加自己的原创节点
   → 补充个人见解和应用场景
```

**示例**：
```
问题："什么是逆否命题？"
已达到：85分（绿色）
    ↓
调用 four-level-explanation
    ↓
学习Level 3-4（专家和创新层次）
    ↓
添加节点："逆否命题在编程逻辑中的应用"
```

子问题5.2："逆否命题等价性的例子？"

## 🛑 何时停止迭代？

### 量化停止标准

建议在满足以下**至少3个**条件时停止迭代：

| 标准 | 目标值 | 说明 |
|------|--------|------|
| **绿色占比** | ≥ 80% | 80%以上的问题已完全理解 |
| **节点数量** | 接近原白板的50-70% | 检验白板已具相当复杂度 |
| **file节点数量** | ≥ 3个 | 至少生成3个解释文档 |
| **原创节点数量** | ≥ 2个 | 至少添加2个个人理解节点 |
| **无红色节点** | 0个红色 | 没有完全不懂的问题 |

### 定性停止标准

**学习感受**：
- ✅ 我能够不看资料，流畅解释所有检验问题
- ✅ 我能够举出自己的例子（不是照搬原白板的）
- ✅ 我能够区分易混淆概念
- ✅ 我能够应用到新场景
- ✅ 我感觉已经"复现"了原白板的知识结构

**避免过度迭代**：
- ⚠️ 不追求100%绿色（80-90%即可）
- ⚠️ 不无限添加节点（50-70%原白板复杂度即可）
- ⚠️ 不过度完美主义（理解>记忆）

### 分阶段目标

**第1轮迭代目标**：
- 绿色占比：30-40%
- 识别出完全不懂的问题并拆解
- 至少生成1个解释文档

**第2轮迭代目标**：
- 绿色占比：60-70%
- 处理紫色节点（似懂非懂）
- 至少生成2-3个解释文档

**第3轮迭代目标**：
- 绿色占比：80-90%
- 添加原创节点
- 建立知识关联

**最终目标**：
- 绿色占比：≥80%
- 检验白板复杂度接近原白板
- 能够独立复现知识结构

## 💡 最佳实践

### Do's（推荐做法）

✅ **渐进式填写**：
- 不强求一次性填写所有黄色节点
- 先填写自己有把握的
- 从简单到困难逐步推进

✅ **充分利用拆解**：
- 遇到完全不懂的，立即拆解
- 不要硬撑，拆解是降低难度的最佳方法
- 拆解后的子问题更容易攻克

✅ **评分反馈驱动**：
- 每次填写后立即评分
- 根据评分反馈选择下一步行动
- 不猜测，让数据说话

✅ **多样化使用解释Agent**：
- 不同类型的理解不足，用不同的Agent
- 易混淆概念→comparison-table
- 需要记忆→memory-anchor
- 需要深度理解→clarification-path

✅ **添加原创节点**：
- 补充你的原创理解
- 建立与其他主题的关联
- 添加你的学习心得

✅ **定期回顾**：
- 每轮迭代后统计绿色占比
- 观察白板的"生长"过程
- 感受知识网络的构建

---

### Don'ts（避免做法）

❌ **不要照抄原白板**：
- 检验的目的是复现，不是复制
- 用自己的话输出
- 即使不准确，也比照抄更有价值

❌ **不要跳过评分**：
- 不能自己觉得"差不多就行"
- 必须用评分系统验证
- 评分反馈会指出你看不到的盲点

❌ **不要畏惧红色**：
- 红色是正常的学习状态
- 红色意味着你找到了盲点
- 拆解和解释会帮你攻克红色节点

❌ **不要过度完美主义**：
- 不追求100%绿色
- 不无限扩展白板
- 达到80%绿色即可进入下一主题

❌ **不要孤立学习**：
- 建立节点之间的连接
- 添加与其他主题的关联
- 思考概念之间的联系

❌ **不要忽略紫色节点**：
- 紫色是"似懂非懂"的危险状态
- 必须重新学习并达到绿色
- 不解决紫色节点会留下隐患

---

### 效率提升Tips

**Tip 1：批量操作**
- 一次性填写多个黄色节点，再批量评分
- 一次性拆解多个难题
- 减少上下文切换

**Tip 2：标记优先级**
- 优先攻克核心概念（原白板的关键节点）
- 次要概念可以先搁置
- 绿色节点优先级降低

**Tip 3：时间分配**
- 第1轮迭代：30-40分钟（初次尝试）
- 第2轮迭代：20-30分钟（针对紫色节点）
- 第3轮迭代：15-20分钟（查漏补缺）
- 总计：60-90分钟完成一个主题

**Tip 4：利用file节点**
- 解释文档可以反复查阅
- 在Obsidian中做笔记关联
- 建立个人知识库

**Tip 5：定期回顾**
- 每周回顾检验白板
- 重新填写黄色节点验证记忆
- 保持长期记忆

## 📊 检验白板复杂度追踪（可选）

### 为什么追踪复杂度？

追踪检验白板的"成长"可以：
- 量化学习进度
- 判断何时完成学习
- 激励持续迭代
- 对比与原白板的差距

### 追踪指标

**基础指标**（推荐追踪）：

| 指标 | 初始值 | 目标值 | 说明 |
|------|--------|--------|------|
| 节点总数 | 10-15 | 30-40 | 白板复杂度 |
| 绿色节点数 | 0 | 7-9 | 完全理解的问题 |
| 绿色占比 | 0% | ≥80% | 学习完成度 |
| file节点数 | 0 | 3-5 | 解释文档数量 |
| 原创节点数 | 0 | 2-5 | 个人理解节点 |

**高级指标**（可选追踪）：

| 指标 | 说明 | 计算方法 |
|------|------|---------|
| 平均评分 | 所有问题的平均分数 | Σ(各问题分数) / 问题数 |
| 层级深度 | 拆解的最大层数 | 从检验问题到最深子问题的层数 |
| 连接密度 | 节点连接的紧密度 | 边数 / 节点数 |
| 迭代轮次 | 完成多少轮迭代 | 手动记录 |

### 追踪方法

**方法1：手动记录（推荐）**

在检验白板的蓝色说明节点中添加追踪表格：

## 🎓 学习理念

### 检验白板的教育哲学

**1. 主动输出 > 被动输入**
- 费曼学习法：能输出才是真理解
- 检验白板强制你输出
- 输出暴露盲点

**2. 迭代深化 > 一次性学习**
- 知识需要反复加工
- 每轮迭代都加深理解
- 从红色→紫色→绿色的渐进过程

**3. 结构化 > 碎片化**
- 知识不是孤立的点
- 通过拆解和关联构建知识网络
- 最终复现完整知识体系

**4. 数据驱动 > 感觉驱动**
- 用评分系统量化理解程度
- 用绿色占比判断完成度
- 不靠"感觉"，靠数据

## ❓ 常见问题

**Q1: 检验白板和原白板有什么技术区别？**
A: 没有任何技术区别。检验白板只是一个后缀带"检验白板"的.canvas文件，所有原白板能做的操作，检验白板都能做。

**Q2: 我必须让所有节点变绿吗？**
A: 不必须。80-90%的绿色占比已经非常优秀。个别难点保持紫色也可以接受。

**Q3: 我可以在检验白板上手动添加节点吗？**
A: 完全可以！鼓励你添加原创节点、补充问题、学习心得等。

**Q4: 迭代多少轮才算完成？**
A: 没有固定轮次。通常3-5轮迭代后，绿色占比会达到80%，此时可以停止。

**Q5: 如果我一个问题都不会怎么办？**
A: 立即调用basic-decomposition拆解所有问题，降低难度后逐个击破。

**Q6: 评分不准确怎么办？**
A: 评分系统基于AI，可能有误差。如果你确信理解正确，可以手动将节点变绿（但不推荐）。

**Q7: 检验白板会变得很大吗？**
A: 通常不会。检验白板最终节点数约为原白板的50-70%，文件大小<100KB。

**Q8: 我可以同时有多个检验白板吗？**
A: 可以。每个主题可以生成独立的检验白板，互不干扰。

**Q9: 检验白板可以保存多久？**
A: 永久保存。检验白板就是普通的.canvas文件，保存在你的笔记库中。

**Q10: 我可以分享检验白板吗？**
A: 可以。检验白板是标准的JSON Canvas格式，可以分享给其他使用Canvas学习系统的用户。
