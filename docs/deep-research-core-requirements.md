## 当前核心问题

### 问题一：规范驱动开发成功率过低 / 返工率过高

我在该项目上分别使用了 BMAD V6 和 Super Power 两套工作流进行规范驱动开发，
但实际执行效果存在严重问题：

| 症状 | 描述 |
|------|------|
| 功能实现与预期严重偏差 | 每次实际实现的功能与 Claude Code 声称的预期相差甚远 |
| 高返工率 | 每次执行 BMAD V6 的 EPIC Story 或 Super Power 的 Plan 计划后，产出代码都需要反复返厂修复 |
| 大量 Bug | 每次开发或修复功能后都会引入一堆新 Bug，形成恶性循环 |

这非常反直觉——规范驱动开发的本意是提高质量、降低返工，但目前的结果完全相反。

### 问题二：Graphiti 记忆检索质量不足

| 症状 | 描述 |
|------|------|
| 记忆累赘 | 积累的记忆过于冗余，导致检索时无法找到核心决策 |
| 时序图未区分优先级 | 时序图没有区分重要信息与次要信息 |
| 过期决策未处理 | 部分已经过期的决策仍然被检索并影响开发方向 |
