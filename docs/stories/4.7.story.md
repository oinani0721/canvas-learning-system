### 检验白板学习循环（8步）

[Source: docs/prd/FULL-PRD-REFERENCE.md Epic 4 Story 4.7]

检验白板的持续扩展机制：

**用户可以进行的操作：**
1. **填写个人理解**：在黄色节点输出自己的理解
2. **发现不足**：意识到某个问题理解不够
3. **继续拆解**：调用拆解Agent拆解子问题
4. **补充解释**：调用6种解释Agent之一
5. **评分验证**：评分系统给出反馈
6. **颜色流转**：红色→紫色→绿色
7. **添加自己的节点**：补充原白板没有的理解
8. **构建完整知识网络**：最终接近原白板的结构

**学习循环示意图**：
```
检验问题（红色）
  ↓ [尝试输出]
  ↓ [发现不足]
  ↓ [拆解/解释/评分]
  ↓ [再次输出]
  ↓ [逐步完善]
  ↓
复现原白板结构
```

### 颜色流转路径

[Source: docs/architecture/tech-stack.md#颜色系统]

**颜色编码**：
- `"1"` = 🔴 红色（不理解/未通过）
- `"2"` = 🟢 绿色（完全理解/已通过）
- `"3"` = 🟣 紫色（似懂非懂/待检验）
- `"6"` = 🟡 黄色（个人理解输出区）

**颜色流转规则**（在检验白板上与原白板完全一致）：
- 初始状态：红色问题节点 + 黄色理解节点（空）
- 用户填写黄色节点后：
  - 调用scoring-agent评分
  - 评分 ≥ 80分 → 红色节点变为绿色
  - 评分 < 80分但有部分理解 → 红色节点可能变为紫色
  - 评分很低 → 保持红色，建议拆解或调用补充解释
- 紫色节点：再次输出+评分后可转为绿色
- 绿色节点：表示完全理解，不再需要扩展

### 可用的Sub-agents清单

[Source: .claude/PROJECT.md#Sub-agents架构设计]

**检验白板上可用的11个Agent**（已实现）：

**拆解系列**：
1. basic-decomposition - 基础拆解（2-7个子问题）
2. deep-decomposition - 深度拆解（复杂概念）
3. question-decomposition - 问题拆解（生成检验型问题）

**解释系列**：
4. oral-explanation - 口语化解释（800-1200字）
5. clarification-path - 澄清路径（1500+字，最详细）
6. comparison-table - 对比表（易混淆概念）
7. memory-anchor - 记忆锚点（类比、故事、口诀）
8. four-level-explanation - 四层次解答（新手→专家→创新）
9. example-teaching - 例题教学（完整解题教程）

**评分**：
10. scoring-agent - 评分（4维度：准确性、形象性、完整性、原创性）

**Canvas操作**：
11. canvas-orchestrator - 主控Agent（协调所有操作）

**何时使用哪个Agent的决策树**（需要在文档中详细说明）：
- 完全不懂（红色） → basic-decomposition（降低难度）
- 似懂非懂（紫色） → scoring-agent（找出问题）→ clarification-path（深度解释）
- 需要记忆 → memory-anchor（生动类比）
- 易混淆概念 → comparison-table（结构化对比）
- 需要练习 → example-teaching（例题）

### 检验白板"生长"过程示例

[Source: docs/prd/FULL-PRD-REFERENCE.md Epic 4 Story 4.6]

**阶段1：初始框架（Story 4.4生成）**
```
检验白板初始状态：
- 1个蓝色说明节点
- 5个红色检验问题
- 5个黄色理解节点（占位符，用户未填写）
- 总计：11个节点
```

**阶段2：第1轮迭代**
```
用户操作：
1. 填写3个黄色节点（尝试输出）
2. 调用scoring-agent评分
3. 1个问题变绿（完全理解）
4. 2个问题保持红色（理解不足）→ 调用basic-decomposition
5. 拆解生成4个子问题 + 4个黄色节点

检验白板状态：
- 11个初始节点 + 8个新增节点 = 19个节点
```

**阶段3：第2轮迭代**
```
用户操作：
1. 填写4个新的黄色节点
2. 调用scoring-agent评分
3. 2个问题变绿，2个变紫色（似懂非懂）
4. 对紫色节点调用oral-explanation
5. 生成2个解释笔记文件，创建2个file节点链接

检验白板状态：
- 19个节点 + 2个file节点 = 21个节点
```

**阶段4：第3轮迭代**
```
用户操作：
1. 阅读解释笔记后重新填写紫色节点的黄色理解
2. 调用scoring-agent评分
3. 2个紫色节点变绿
4. 用户添加1个自定义节点（自己的原创理解）

检验白板状态：
- 21个节点 + 1个自定义节点 = 22个节点
- 颜色分布：5个绿色（通过）+ 2个红色（剩余难点）
```

**阶段5：最终状态**
```
经过持续迭代：
- 检验白板节点数：30-40个（接近原白板的50个节点）
- 绿色节点占比：80%以上
- 结构复杂度：接近原白板（有拆解层次、解释文件、原创补充）
- 学习成果：真正复现了知识体系
```
