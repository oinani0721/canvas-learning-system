## Part 1: 原白板 (5.6/10, 覆盖 55%)

**强**: 批注几乎零摩擦 (选中→2步 modal→callout+理解度, 契合 Obsidian callout 原生能力)。派生节点显式选 7 类关系 (优于原生"有链接没语义")。

**弱 (同一根因)**: 擅长"记录批注"却不擅长"保存剖析过程"。
- Story 2.4 主路径回退成"callout→frontmatter tips[]", Graphiti Plan B 废弃, 批量同步入口打 410 deprecated marker → 批注不升级为"可被检验白板调用的学习记忆"
- 派生关系停留"文件里有字段", plugin 对 /sync/relationships 0 调用, relationship_sync_service.py 实写 Neo4jEdgeClient → 系统看不到"关系怎么被建立/修正/再修正"
- 节点对话仍 clipboard 中转 (规格写"节点页内即可对话无需切窗口", 实际是 clipboard→Claudian) → 日常高频伤心流

**6 维度**: 可发现性 中偏好 / 可回溯性 明显不足 / 可测量性 不足 / 针对性 不足 / 可操作性 中 / 学习效率 尚未达到。

**建议**: P0 批注+关系+对话统一事件化+版本链 (最重要); P1 节点对话原生化; P1 关系类型+描述+路径上下文纳入可检索语义。

## Part 2: 检验白板 (4.2/10, 覆盖 38%)

**强**: 学习科学方向比实现成熟 — 独立目录/exam_board 类型/防嵌套/防泄露/空白 UI/弱点选题/提示跳过/校准/回写, reference_answer 强制 None。

**核心问题**: "针对性"目前是**引语层针对性**, 不是**路径/错误模式/关系推理/记忆时序层针对性**。
- MVP-α 从 frontmatter.tips 取批注, prompt 强制引用批注原话, 题目存内存 ring buffer (重启即丢), 评分按 question_id 回读题面
- "感觉有点针对"因为引用了你的话, 但你要的是"精确多段推理节点关系 + 理解度 + 错误 + 批注, 让我想起原白板内容, 再考是否犯相似错误"
- LITE-4-3 修正版已意识到 (补 wikilink 邻居进主路径 + question_text 持久化 + 评分回读真实题面), 方向对但未变稳定产品
- hint/skip/校准设计很像好的考试行为系统 (4 级提示+不惩罚跳过+提示影响权重+投票回写), 但 plugin 0 调用 → 用户走不进去

**6 维度**: 可发现性 中 / 可回溯性 明显不足 (内存 store 重启丢) / 可测量性 不足 (题面漂移风险) / 针对性 **核心不足** / 可操作性 中偏低 / 学习效率 尚未达到。

**建议 (都 P0)**: ①出题上下文升级为事件溯源 (最近批注+错误修正+显式关系+1-2跳邻居+路径描述); ②评分对象永久绑定真实题面 (否则 FSRS/BKT 全是噪音)。

## Part 3: 个人记忆系统 (设计 7/10, 当前产品支撑 1/10)

**方向对**: 后续 spec 已正确拆职责 — LanceDB 精确笔记片段, Graphiti 学习事件+关系事实; CanvasGraphEpisodeV1 统一接 callout/wikilink/calibration/error; belief_key 维护认知命题版本链; GraphitiRelationService 收敛检索成产品语义 API。episode 带 path_trace/in_links/out_links/narrative + belief version chain 支持"我以前认为 X 后来改成 Y" → 比单纯存 tips 高一级。

**坏消息**: 真正流进脊柱的事件几乎没有 — 批注/对话/Edge 三管道全断, Graphiti 引擎活着但通道全断, MVP-α 只读 frontmatter.tips 凑表层。

**与 FSRS/Anki 关系**: Graphiti 价值不是替代 FSRS, 而是给 FSRS 提供更语义化/可解释的记忆证据层 (为什么这节点薄弱、和哪些路径/错误/批注有关、这次该追哪种关系型题)。

## Part 4: 对标 Obsidian 生产力标准

- 原白板: 比原生多"显式关系类型+节点派生+学习状态字段", 但 Breadcrumbs (typed links 遍历)/ExcaliBrain (链接+Dataview+frontmatter+反链解释成可导航脑图) 已很成熟 — Canvas typed-link 价值未释放 (没进统一记忆+出题主路径)
- 检验白板: 理论优势更强 (剖析+自测同环境零切换, 不用切 Anki), 但 Anki/FSRS 生产力 = 调度可靠+目标 retention 显式+工作量可预测+每卡记忆状态可解释; Canvas 在"上下文丰富"有潜力, 在"调度可靠/评分可信/长期稳定"明显弱
- Matuschak"prompt 质量决定闭环"在此是生死线: 题目太泛→退化泛概念测试; 不绑定原话/路径/错误→不可能"再次想起原白板内容"。LITE-4-3 方向对, 但提示语不能只"引用一句原话", 要"同时指明你当时在哪条路径卡住/当时怎么写/现在要区分什么"

## Part 5: 核心闭环断点修复优先级 (按学习效果 ROI, 非技术快慢)

| 优先级 | 修复项 | 为什么高 ROI |
|---|---|---|
| **P0** | 真实题面持久化+评分回读 + wikilink 邻居进出题主路径 (V-08+V-10) | 直接修"针对性是假的"+"测量信号会脏"两大致命伤 |
| **P0** | 统一学习事件入口 + belief version chain | "个人记忆系统"从口号变能力的前提 |
| **P1** | hint/skip/calibration/新疑问回流 接前端 | 决定真实考察能否继续推进并形成闭环 |
| **P1** | 节点对话原生化 | 提升原白板日用频率与深挖质量 (但内容须进记忆脊柱才有长期价值) |
| **P2** | Dashboard 学习链路可视化 | 提升回顾/自我管理, 不是第一闭环断点 (须建在可信数据上) |

## Part 6: 设计哲学自洽性

| 哲学 | 是否成立 | 最大不自洽点 | 改法 |
|---|---|---|---|
| 答题=打批注 | ✅ 成立 (最有辨识度) | 答题产物未稳定转成下轮可调用记忆 | 答案/新疑问/纠错都事件化回写节点档案 |
| 完全静默评分 | ⚠️ 部分成立 | **MVP-α 实际保存后追加 "## 反馈 (x/5)" + 弹 Notice = 延迟显式评分** (Claude 已核实 exam-quick.ts:146) | 改"默认静默+可按需展开+状态先行": 保存只看轻量状态标记, 详细反馈默认折叠/一轮后集中回看, 分数先进节点 profile 不占答题现场 |
| 永久沉淀 md | ✅ 完全成立 (相对外部卡片最大长期资产) | 沉淀了但未充分结构化利用 | 统一 block 引用+question_id+belief_key+source trace |

## Part 7: 一句话判定 + 唯一先手

> 如果现在就推到"生产力标准高质量使用", 最该先做的不是再加功能, 而是**把原白板里的批注、双链关系、节点对话与错误修正统一写成同一条可追溯的个人记忆脊柱, 然后让检验白板只围绕这条脊柱出题、评分、回写**。

**唯一正确先手 (可执行指令)**:
```
批注/关系/对话/校准 → 统一事件化 → Graphiti 版本链 + LanceDB 片段
  → 真实题面持久化 → relation-aware 出题 → 评分回读 → 节点档案回写
```
在这条主干成形前, 不要优先放精力在更多 UI 花样上。
