# 原白板 + 检验白板 设计层审查任务书

## 0. 一句话任务

**从设计层面审查 Canvas Learning System 的两大核心功能 —— 原白板 (知识剖析) + 检验白板 (主动回忆考察) —— 能否达到"生产力标准、高质量使用"。着重聚焦用户在实际学习过程中对这两个功能的真实需求, 而不是技术细节是否优雅。**

## 1. 项目背景

Canvas Learning System 原本是 Tauri 桌面应用, 2026-04 降级到 **Obsidian Hybrid 部署** (Obsidian 编辑器 + plugin + FastAPI sidecar + Claudian Skill + Neo4j/LanceDB/Graphiti)。用户是 CS 61B 学生, 用它做个人化学习。

两大核心功能 (PRD §1.1 "灵魂二分法", d=1.50):
- **原白板 (剖析模式)**: 信息全可见。用户拖双向链接拆分/连接知识节点、打批注、跟 AI 对话深挖、拉出新概念。用户 80% 时间在此。
- **检验白板 (Active Recall)**: 信息全隐形。AI 用你原白板的批注/错误/关系针对性出题, 你在 md 编辑器手写答案 (哲学: "答题就好比打批注"), 后台静默评分。Karpicke 2011 检索练习 d=1.50, 用户锁定"不可妥协 100% 等价"。

## 2. ⭐ 用户对这两个功能的真实学习需求 (全部原话, 这是审查灵魂)

### 2.1 核心母题 (2026-05-13)
> "我个人极其关注的是**我的批注**, 以及我在原白板使用**双向链接**来**拆分和联系节点**的**整一个探索的过程**, **批注是核心**, 我需要用我的**个人记忆系统**充分的理解我使用原白板的学习过程, 这样我使用**检验白板**的时候, 才可以根据我的**原白板学习过程**来**极其针对性**的考察我。"

### 2.2 用原白板时的真实场景
- "我在 obsidian 上是用 md 文件 + **自己定义双向链接来规划各个节点之间联系**"
- "拉出新节点的目的是**聚焦于这块内容进一步讨论**"
- "我对 md 节点内容打下的批注, **这个批注是我的核心想法也是我后续需要聚焦考察的点**"
- "我用双向链接连接节点时, 有让我**填写节点和节点之间是什么关系**, 但 claude code / claudian 如何能正确识别则是一个问题"
- "AI 讲解时能**精确返回笔记片段**"

### 2.3 用检验白板时的真实场景 (最完整原话)
> "在使用原白板生成检验白板时……可以**精确的多段推理各个节点之间的关系**, 然后能理解到我各个节点……**标记的理解程度**, **各个节点犯下的错误**, 以及**各个节点我自己打下的批注**, 结合以上节点考察我, **让我想起了原白板的内容**, 并且**再次考察我是否会犯下原白板中相似的错误**。"

- "检验白板考察时我们是如何**一来一回交互**的, 如果我对题目**卡壳又该怎么办**?"
- "我的所有回答, **全部停留在这个 md 上吗?**"
- "我已经**抛弃了计时功能**" (明确否决)
- "你出题后我记得有**跳过等选项**给我选择"
- "检验过程暴露的**新疑问要作为新点, 下次检验白板能考到**, 我希望**点击疑惑能跳转看到当时解答探究的过程**"

### 2.4 对"生产力标准/高质量使用"的期望 (用户没说"生产力"但表达了同义门槛)
- **静默不打断心流**: "隐形评分设计的初衷是什么?" → 锁定"立即评分+完全静默"保护 Flow State
- **思考永久沉淀**: 答题=打批注、"全部停留在 md"
- **节点 3 层 profile 可追溯**: "点击 node 能看到考察记录/错误历史/tips/掌握情况"
- **Dashboard 看见学习链路**: "能知道用过的原白板, 对应产生过哪些检验白板, 原白板剖析知识点的链路怎么呈现"

### 2.5 最高优先架构声明
> "检验白板的设置, **Graphiti 的记忆系统 (这是最重要的架构设置)**, 以及个人笔记返回系统。"

### 2.6 关键认知校准 (用户纠正过 Claude)
Claude 曾把检验白板误解为"通用空白考察环境", 被用户批注修正: 真相是"题目会**显式引用你之前的批注**('你之前说过 X, 但其实应该 Y'), 不是通用题库"。

## 3. 当前实现真相 (9-agent workflow 实证, 供 ChatGPT 对照)

| | 真实完成度 | 关键断点 |
|---|---|---|
| 原白板 | **68%** (声明 80%) | 批注→Graphiti 死代码 (tips.py:309 raise 410); Edge 关系卡 frontmatter 不入库; 节点对话是 clipboard hack 原型 |
| 检验白板 | **42%** (声明 45%) | hint/skip/校准后端齐全但 plugin 0 调用; V-10 评分对象漂移; V-08 wikilink 不进出题路径 |
| **用户核心闭环** | **37.5%** | Graphiti 三管道全断 (批注/对话/Edge), 个人记忆系统真实落地 ≈ 0% |

**MVP-α 是唯一端到端能跑的检验白板路径** (exam_quick + exam_grade), 但靠"直读 frontmatter.tips 绕过 Graphiti"凑表层针对性。

## 4. 6 个设计层审查议题 (请聚焦学习需求, 非技术细节)

### 议题 1: 原白板"剖析模式"设计能否支撑用户的真实剖析行为?
用户要"拖双链拆分/连接节点 + 打批注 + 拉新节点聚焦深挖 + 填写节点关系"。当前设计 (ai-linked-doc 7 类关系 + callout 批注 + clipboard 节点对话) 在**学习体验层面**是否流畅? 缺什么让用户"愿意每天用"?

### 议题 2: 检验白板"针对性考察"设计能否兑现用户原话?
用户要检验白板"精确多段推理节点关系 + 理解度 + 错误 + 批注, 结合考察, 让我想起原白板内容, 再考是否犯相似错误"。当前设计 (MVP-α 直读 tips 出题 + 静默评分) 离这个目标多远? "多段推理节点关系"在设计上如何实现?

### 议题 3: 个人记忆系统 (Graphiti) 作为"最重要架构"的设计是否到位?
用户把 Graphiti 列为最高优先。当前三条管道全断。从**设计层面**, Graphiti 应如何串起"原白板批注/探索 → 个人记忆 → 检验白板针对性出题"这条用户母题脊柱? (已有 epic-5a-graphiti-runtime 5 spec, 评估其设计是否对路)

### 议题 4: 两白板的"同环境零切换"是否构成相对 Obsidian+Anki 的生产力优势?
对标 Obsidian 社区 (剖析用 Breadcrumbs/Excalibrain, 自测用 Spaced Repetition 插件需切 Anki)。Canvas 双板同在 Obsidian 是结构性优势, 但要兑现需要什么? Matuschak "prompt 质量决定闭环成败" 如何应用到检验白板出题?

### 议题 5: 用户核心闭环 (37.5%) 的设计断点优先级
完整闭环: 原白板拆解 → AI 对话加深 → FSRS 定复习 → 检验白板隔离考察 → 静默评分更新掌握度 → 错误 Day3/7 追踪。哪个断点最该先补 (从学习效果 ROI, 不是技术难度)?

### 议题 6: "答题=打批注" + "完全静默评分" + "永久沉淀 md" 的设计哲学是否自洽?
用户锁定这三条。它们在设计上是否互相支撑? 静默评分 + md 沉淀对 Flow State 和长期回顾的实际学习价值?

## 5. 期望输出 (7 Part, 中文)

- **Part 1**: 原白板设计成熟度评分 (X/10) + 对用户剖析需求的覆盖度
- **Part 2**: 检验白板设计成熟度评分 (X/10) + 对用户考察需求的覆盖度
- **Part 3**: 个人记忆系统 (Graphiti) 设计是否支撑母题脊柱 + 具体设计建议
- **Part 4**: 对标 Obsidian 社区生产力标准的差距 (按原白板/检验白板分列)
- **Part 5**: 用户核心闭环断点的修复优先级 (按学习效果 ROI 排序)
- **Part 6**: 设计哲学自洽性评估 (答题=打批注 / 静默评分 / md 沉淀)
- **Part 7**: 一句话判定 — 这两个功能要达到"生产力标准高质量使用", 设计层面最该先做什么

## 6. 严格要求
- **聚焦学习需求和产品体验, 不是技术细节是否优雅**
- 必引用 spec / PRD 的 file:line 作证据
