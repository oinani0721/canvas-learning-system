## 第 6 部分: 我们用什么流程开发? (你问的"还用 BMAD 吗")

**用 BMAD 的骨架,不用它的全套仪式:**

| 用 / 不用 | 内容 |
|---|---|
| ✅ **用** | spec 文件格式 (frontmatter + 验收标准 AC + 任务 checkbox) — 多窗口接续靠它 |
| ✅ **用** | sprint-status.yaml 状态追踪 + live/archive 治理 |
| ✅ **用** | R4 循环: 你写期望 → 我自决技术 → ship 给你上手 → 你 UAT 批注 → 我调整 |
| ❌ **不用** | bmad-bmm-dev-story 自动实施 skill (Graphiti 精确 schema 手写更稳) |
| ❌ **不用** | BMAD 的 SM/PO/QA 角色仪式 (早就融入"我自决 + 你批注") |
