## 🔵 批注 ① · 系统怎么记录? Obsidian wikilink + Graphiti 怎么协作?

### 一句话答案
**wikilink = 静态图结构 (永远在 .md 里), Graphiti = 动作记录 (何时怎么学)**. 二者**互补不重复**.

### 你写一条批注的完整生命周期

```
你在 Obsidian 写 "[!tip]+ 我对 base case 不理解"
                ↓
┌─── Sprint 1 已实现 ────────────────────────┐
│ 第 1-2 秒: Obsidian 端                     │
│ ① metadataCache.on('changed') 触发        │
│ ② FrontmatterTipsSync 自动写入            │
│ ③ frontmatter.tips[] (含时间戳)            │  ← 永久档案
│ ④ 状态栏 "🎓 N 条 Tips" 更新              │  ← 你能看见的反馈
└────────────────────────────────────────────┘
                ↓ 数据在 .md 文件里, 还没进后端
                
┌─── Sprint 1 末出题时 (Cmd+Shift+Q) ─────────┐
│ 第 3 秒: Backend                            │
│ ① 读 .md frontmatter.tips[]                │
│ ② 拼 prompt: "用户批注'XXX'..."            │  ← AI 真用你的原话
│ ③ 生成题目                                 │
└─────────────────────────────────────────────┘
                ↓
                
┌─── ❌ β-1 才补 — 当前 Graphiti 零接入 ──────┐
│ ① POST /api/v1/event/navigation 上传       │
│ ② Graphiti.add_episode(                    │
│     "user 在 base-case 写 tip @ t0")       │  ← 时序记忆
│ ③ 下次出题: search_memory_facts(...)       │  ← 含"你 3 天前从 X 学到 Y"
└─────────────────────────────────────────────┘
```
