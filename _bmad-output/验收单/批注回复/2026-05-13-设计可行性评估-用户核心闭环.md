## 1️⃣ 5 个维度详细评估（每个 Agent 一个维度）

### 维度 A：批注闭环可行性（Agent 1）→ **FAIL**

**核心发现**：3 处管道断在同一链，任一处独立修复都无效

| Gap                          | 严重度      | 现状                                                                                                       |
| ---------------------------- | -------- | -------------------------------------------------------------------------------------------------------- |
| G-A1 Cmd+Shift+A 不入库         | Critical | `main.ts:2023-2035 handleAnnotateCallout` 整段 0 fetch / 0 POST                                            |
| G-A2 7 类 callout 类型路由        | High     | 后端 `apply_external_signal` 只接受 misconception/problem_trap/guided_thinking_correct 3 种，与用户 7 类不对齐         |
| G-A3 理解度 checkbox 不进 mastery | High     | `/mastery/{id}/self-assess` endpoint 存在但 plugin 不调                                                       |
| G-C1 批注归属节点 schema 错位        | Critical | `question_generator.py:917-944` 查 `e.node_id`（顶层属性），但 `memory_service.py:1239` 写在 metadata 字典里           |
| G-C2 callout → BKT 触发不存在     | Critical | `mastery_engine.apply_external_signal` 算法 ready，但 plugin 无调用方                                            |
| G-E2 exam board 不读 callout   | Critical | `_get_error_history` 查 `source_description='error_record'` 全表零 writer（writer 写的是 'misconception-record'） |
| G-E3 node_id schema 三套写法不一致  | Critical | metadata 字典 / 顶层属性 / agent-{timestamp} 三套，查询永远命中不了                                                       |

**修复工作量估算**：4-5 个 Story 联动（Story 2.4 callout + Story 5.1 mastery + 新 source_description schema + 新 node_id 顶层 property + question_generator 改造）= **80-120 小时**

### 维度 B：双向链接探索过程记忆化（Agent 2）→ **FAIL**

**核心发现**：5 个能力（时序/意图/路径/注意力/演化）**全部缺失**

| 能力 | 状态 | 关键证据 |
|---|---|---|
| 时序记忆（链接 A→B 何时建立） | ❌ 缺失 | `wikilink_graph_service.py:226-230 refresh()` 是全量 rebuild，无 per-edge timestamp |
| 意图记忆（为什么建链接） | ❌ 缺失 | 整个 codebase 无 `intent` / `reason` / `motivation` 字段 |
| 路径记忆（A→C→B→D 浏览） | ❌ 缺失 | Plugin grep `active-leaf-change|file-open|editor-change` = **0 命中** |
| 注意力记忆（停留多久） | ❌ 缺失 | grep `dwell|attention|view_count` = 0 命中。`BehaviorTracker` 在 known-gotchas 提到但 codebase 找不到（ghost code） |
| 演化记忆（链接修改历史） | ❌ 缺失 | `refresh-changed` 不调 `record_temporal_event`，wikilink 增删 0 事件进 Graphiti |

**典型场景失败**：
- 你 Cmd+Click `recursion → base-case → factorial` 探索 → 后端**完全不知道**这条访问路径发生过
- 你在 `recursion.md` 停 15 分钟读 → 后端**零捕获**
- 你把 `[[X]]` 改成 `[[X|alias]]` → WikilinkGraph 全量 rebuild 覆盖，历史丢失

### 维度 C：Graphiti 充分理解学习过程（Agent 3）→ **PARTIAL**

**核心发现**：schema 层支持温度演化 + verifiability，但**内容层失效**

| 问题            | 真相                                                                                                                                      |
| ------------- | --------------------------------------------------------------------------------------------------------------------------------------- |
| Episode 内容粒度  | `memory_service.py:530-540` 模板：`"Student learned 'X' using Y agent on canvas Z. Node: N (score: S/100)"` — 单句行为日志，不含对话原文 / 解题步骤 / AI 解释 |
| LLM 抽取 prompt | `graphiti-core/prompts/extract_nodes.py:77-122` 是**通用** "extract entities from text" — 无教学场景定制                                          |
| Dedupe 破坏演化   | `dedupe_nodes.py` 把 5/1 学 recursion + 5/13 学 recursion 合并成同一 entity，description 被后写覆盖，**演化痕迹被擦除**                                       |
| 时间感知能力        | edge invalidation 完全依赖 LLM 在 prompt 里看到否定句（"stopped"），单句日志根本不会出现这种语义 → 老边永远不被 expire                                                    |
| 跨概念推理         | `Misconception` schema 字段是 description/error_type/remedy/node_id，**没有 root_cause_concept 字段** — 无法表达"搞混 A 是因为 B 没学好"                    |
| 检索回溯          | `edge.episodes: list[str]` 字段在 schema 层存在（可回溯原文）但 `memory_service._search_graphiti:1474-1517` 解析时**丢弃这个字段** → 前端拿不到                     |

**关键诊断**：营销话语 "temporal knowledge graph" 在 schema 层为真，**在内容层失效**。
