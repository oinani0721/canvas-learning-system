## 7️⃣ 关键反直觉发现（★ Insight 汇总）

`★ Insight ─────────────────────────────────────`
- **"完成 1 个 Epic" ≠ "用户能用 1 个 Epic"**：Epic 1 v2 是唯一 17/17 done 的 Epic，但**所有功能都是基础设施层**（vault / wikilink / 4 MVP UI）。用户能产生批注（1.16 UI 100%）但批注不入库（G1）、能看到 Dashboard（1.18 v1）但显示假数据。**所谓 done 是工程粒度的完成，不是产品价值的释放**。
- **Epic 5 / Epic 6 是产品核心价值载体但 0 启动**：用户原话"极其针对性考察我"的精准载体（Epic 4 检验白板 11 stories + Epic 5 精通度追踪 8 stories）全 backlog，**0% 兑现**。这两个 Epic 之外的所有工作（Epic 1 基础设施 / Epic 2 智能对话 / Epic 7 复习 / Epic 8 Dashboard）都是为它们做铺垫。
- **算法 100% / plugin 0% 是 Epic 5/6/7 的共同特征**：Backend 已建好 2893+ 行算法/服务/MCP/REST 产线（mastery_engine 778 + mastery_fusion 216 + signal_registry 263 + question_generator 1149+ + exam_service 487+），但 **plugin 端只调一个孤儿 endpoint `/api/v1/exam/start`**。10 个 mastery REST + 3 个 mastery MCP tools 是 100% 孤儿 API。
- **sprint-status.yaml 编号双轨是工程治理 P0**：Epic 4↔6 互换 + Epic 5 命名 8/8 错配 + Epic 3 spec 12/13 缺失 — dev-story Skill 永远找不到正确 spec。**修这 10 min 比新写任何 Story 优先级都高** — 它直接阻塞所有后续 dev 工作的 BMAD 工具链。
- **本 session 8 commits 是"读路径修完，写路径未动"的精准定位**：P0-5 修了 exam board 读 tips/errors 的 schema 契约 / Story 2.3 修了 chat 读历史误解的 path / Wave-5 follow-up 修了 vault_id 注入。**但用户产生数据进系统的写路径（G1 callout 入库 / G2 plugin extract 触发 / G4 plugin 调 mastery）一处未修**。这是 ChatGPT A+ 路径 next session 的核心。
- **Story 1.18 Dashboard 与 Epic 8 v2 的鸿沟**：用户看到 `mastery: 0.30` 是 plugin 派生节点时硬编码的死值（`node-derivation.ts:mastery_score:0.3`），不是 BKT 真值。**Epic 8 v2 8 stories 全 backlog 的根因不是 v2 没设计好，而是 Epic 5 mastery 真值从未写入 frontmatter（G3）**。修 Epic 8 v2 前必修 Epic 5 plugin hook（P0-6）。
