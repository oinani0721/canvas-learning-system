# 功能开发进度全景对照

> [!info]+ 这是什么
> 把"全景文档的 6 个维度 + 7 大架构断层"与"BMAD Sprint 的 Story 实施进度"**逐一对照**，告诉你哪些 Story 标 done 但代码只完成了一半、哪些算法 production-ready 但 plugin 不调用、哪些 spec 标 ready-for-dev 但 backend 已 100% 实施。

## 0️⃣ 一句话总览

> **基础设施 100% production-ready，但用户感知能力 5-30%** —— **写路径**（用户产生数据 → 进系统）和**读路径**（系统取数据 → 喂 AI）之间断三处。本 session 7 commits 修了所有"读路径"（Story 2.3 + P0-5 source_description），但"写路径"（G1 callout 不入库 + G2 post-turn-extract 不触发 + G4 plugin 不调 mastery MCP）**一处未修**。

## 6️⃣ 关键反直觉发现（★ 重点）

`★ Insight ─────────────────────────────────────`
- **算法层 100% / 用户层 0% 是当前最大鸿沟**：BKT/FSRS/Fusion 全 production-ready，但 vault 里 `mastery: 0.30` 永远是 plugin 派生节点时硬编码的死值（`node-derivation.ts:mastery_score:0.3` + `configure-whiteboard.ts:'mastery: 0.30'`）。修 3 行 plugin hook（callout → backend `/mastery/graphiti-sync`）就能让算法第一次被用户看到。
- **Story 标 done 不等于代码完成**：Story 1.16 标 done 但只完成 frontend UI 部分；Story 1.2 标 done 但 AC #4 增量"假实现"；Story 2.5 标 review 但 backend 100% / plugin hook 0%。**Sprint-status 标 done 是"工程团队认为这步可以告一段落"，不是"用户视角能用"**。
- **Spec 与代码权重不一致**：Story 5.3 spec 写"BKT 0.30 / FSRS 0.25 / 错误历史 0.20 / 校准 0.15 / 自评 0.10"，代码实际"BKT 0.30 / FSRS 0.25 / **ExamScore** 0.25 / Calibration **0.10** / SelfConf 0.10" — 错误历史信号被替换成 ExamScore。**Spec 看起来好 ≠ 代码按 spec 实施**。
- **`edge.episodes` 字段被解析时丢弃**：Graphiti schema 支持 verifiability（每条 fact 边带 episode UUID 列表），但 `memory_service._search_graphiti:1471-1492` 解析时只保留 `uuid/fact/name/created_at`，丢弃 `episodes` 字段。这是 ChatGPT 第二轮指出的 Claude 漏看盲点 — **前端拿不到"AI 这个判断从哪条 episode 来"的回溯链**。
- **G7 raw Cypher 防御就位但未迁移**：wave-5 Stage C 已加 lefthook pre-commit `cypher-vault-filter-lint` 守门，但 30+ 处历史 raw cypher 实际迁移到 `cypher_with_group_filter` 留作 wave-6 backlog。**新代码不能写 raw cypher（hook 拦截），但旧代码的潜在跨 vault 泄漏窗口仍未闭合**。
`─────────────────────────────────────────────────`
