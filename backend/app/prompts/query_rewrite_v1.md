# 查询改写 Prompt（Multi-Query + Decomposition Rewrite）

根据查询的复杂度，有两种改写策略。

## 策略一：Multi-Query 改写（中等复杂度查询）

当用户查询为中等复杂度（包含"如何""为什么"等疑问词，或长度 >= 20 字符），从不同角度改写。

### 输入
- **原始查询**: {{query}}

### 任务
请从不同角度改写以下查询，生成2-3个等价查询。每行一个查询，不要编号，不要解释。

### 输出格式
每行一个改写后的查询，不含编号、标点前缀或任何解释文字。

---

## 策略二：Decomposition 拆分（高复杂度查询）

当用户查询包含连接词（"和""以及""同时"）或多部分结构，拆分为独立子问题。

### 输入
- **原始查询**: {{query}}

### 任务
请将以下复杂查询拆分为2-3个独立的子问题。每行一个子问题，不要编号，不要解释。

### 输出格式
每行一个独立子问题，不含编号、标点前缀或任何解释文字。

---

## 质量要求

1. 改写/拆分的查询必须保留原始查询的核心语义
2. 生成的查询应互相补充，覆盖不同角度
3. 每个查询独立可用，不依赖上下文
4. 支持中英文混合查询
5. 输出行数严格控制在 2-3 行
