# Edge KG-Triplet 提取指令

你需要从用户的解释中提取结构化的 KG-triplet 关系。

## 输入

- `source_concept`：连线起点概念名称
- `target_concept`：连线终点概念名称
- `user_explanation`：用户对两个概念关系的解释（可能是多轮对话的综合）

## 输出格式

请以 JSON 格式输出：

```json
{
  "relation_type": "关系类型的简短标签",
  "rationale_text": "用户原始解释的精炼总结",
  "confidence": 0.0-1.0
}
```

## relation_type 提取规则

从用户解释中提取最准确的关系类型标签。常见类型包括但不限于：

- **因果关系**：是前提条件、导致、引发、决定
- **层次关系**：是特殊情况、是子类、包含、属于
- **对比关系**：相互对比、是替代方案、是反例
- **组合关系**：互相配合、共同构成、组合使用
- **时序关系**：先于、后于、同时发生
- **依赖关系**：依赖于、基于、需要
- **应用关系**：应用于、用来解决、实现了

如果用户的解释不能归入以上类别，请用用户自己的语言创建一个简短标签（不超过 8 个字）。

## confidence 评估规则

| 分值 | 条件 |
|------|------|
| 0.9-1.0 | 用户给出完整的因果解释 + 条件限制 + 用自己的话表述 |
| 0.7-0.8 | 用户给出因果解释但缺少条件限制或反例 |
| 0.5-0.6 | 用户给出表面描述，缺少深层因果 |
| 0.3-0.4 | 用户只说了"有关系"或给出非常模糊的解释 |
| 0.1-0.2 | 用户明确表示不确定或拒绝解释 |

## 注意事项

- relation_type 必须简短（不超过 8 个汉字或 20 个英文字符）
- rationale_text 保留用户的原始表述，不要过度改写
- confidence 反映用户理解的完整度，不是关系本身的正确性
