# Brainstorming / BMAD 决策沟通 + 决策审查闭环

> **用户明确反馈（2026-03-11）：量化细节对决策没有意义，不要让用户做微观技术选型。**
> **用户明确反馈（2026-03-12）：技术解释太专业听不懂。用户是甲方，AI 是乙方——必须用甲方能理解的方式"推销"方案。**

## ⛔ 决策解释强制使用 /深度澄清 — MUST

**当需要用户做任何技术方向决策时，MUST 先调用 `Skill("深度澄清")` 将技术概念翻译为用户可理解的语言。**

### 触发条件（满足任一即触发）
- 向用户展示 2+ 个技术方案供选择
- 询问用户是否认可某个技术方向/算法/架构
- 解释算法原理让用户判断是否符合预期
- 用户主动要求解释某个技术概念

### ⛔ 绝对禁止
- ❌ 未调用 /深度澄清 就直接向用户提技术方向问题
- ❌ 用专业术语堆砌来解释方案（用户不是技术专家）
- ❌ 假设用户能自行理解 API/算法/架构细节

### 正确流程
```
1. AI 内部完成技术调研 → 选出推荐方案
2. ⛔ 调用 Skill("深度澄清") 将方案原理翻译为用户语言
3. 用翻译后的语言向用户展示：选了什么 + 为什么 + 用户体感会怎样
4. 用户确认方向 → 实现 → 用户体验验证
```

## 用户需要的

1. **大方向决策** — 用通俗语言解释的算法原理、架构路径、技术路线
2. **代码/算法实现是否符合预期** — 用户需要理解实现原理，但需要用 /深度澄清 翻译
3. **用户体验验证** — 最终决策是否有效，由用户实际使用体验决定，而非纸面指标

## 用户不需要的（⛔ 禁止提问）

以下类型的问题**不要问用户**，直接选择当前最成熟、最高质量的方案：

- ❌ "bge-m3 sparse 输出替代 BM25+jieba，你觉得可以接受吗？"
- ❌ "Contextual Retrieval 用 Gemini Flash 还是本地模型？"
- ❌ "RRF k=60 还是 k=30？"
- ❌ "chunk size 用 400 还是 512 tokens？"
- ❌ 任何需要用户对比两组数字才能回答的问题
- ❌ 任何未经 /深度澄清 翻译的专业术语堆砌

## ⛔ 决策审查闭环 — 每个决策必须生成验证任务

**核心原则**：决策的有效性不由做决策的 session 自己判定，而是由独立的验证 session 通过严格测试来判定。

### 强制流程

```
决策 session 做出决策 → [立即] 记录 [Decision-Review] → [后续] 独立 session deep explore 制定验收标准 → [实施后] 执行测试 → [结果] 记录 [Test]
```

### [Decision-Review] 记录格式（⛔ 做出决策后必须立即记录）

```
group_id: "canvas-dev"
name: "[Decision-Review] {决策简述} — 待验证"
episode_body: "Session：{来源session} | 决策内容：{选了什么+否决了什么+理由} | 涉及模块：{文件/模块} | 审查维度：{需要验证的关键假设} | 验证状态：PENDING | 验证session：未分配"
```

### 审查维度模板

| 决策类型 | 必须验证的维度 |
|---------|--------------|
| 算法/模型选型 | 在真实数据上的效果、延迟、资源消耗、中文支持 |
| 架构设计 | 数据流完整性、模块间接口、性能瓶颈、扩展性 |
| 前端方案 | 用户体验、加载速度、兼容性、可维护性 |
| 集成方案 | 接口匹配、数据格式、错误处理、降级策略 |

### 验证 session 的职责

1. `search_memory_facts("Decision-Review PENDING")` — 获取所有待验证决策
2. 对每个待验证决策进行 **deep explore 社区最佳实践**
3. 制定 **严格的验收标准**（必须覆盖真实使用场景，禁止合成数据）
4. 设计 **具体的测试用例**
5. 记录 `[Acceptance-Criteria]` 到 Graphiti
6. 实施后执行测试，记录 `[Test]` 结果

### 验收标准的标准

- 不是"纸面指标说好就好"，而是"**用户体验证明好才是好**"
- 测试必须覆盖**真实使用场景**，而非合成数据
- 每个验收标准必须有**明确的通过/失败判定条件**

### 向用户提问的原则

| 可以问 | 不要问 |
|--------|-------|
| "推荐 hybrid search 路线，原理是 X，你认可这个方向吗？" | "RRF k 值选 30 还是 60？" |
| "这个算法的核心逻辑是 X，符合你的预期吗？" | "sparse vs dense 的 nDCG 差 2.3%，你选哪个？" |
| "大方向 A 还是 B？区别是..." | "参数 X=0.3 还是 X=0.5？" |
