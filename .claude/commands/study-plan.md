# 学习计划

基于学生当前的精通度和记忆稳定性数据，制定个性化的复习计划。

**用法**: `/study-plan <概念名称或学习目标>` 或 `/study-plan`（基于当前节点的整体情况）

## 输入适配

用户提供的信息如下：

$ARGUMENTS

请将上述内容解析为：
- 如果有具体概念名，作为 `concept`
- 如果有学习目标描述（如"准备考试"、"巩固基础"），作为 `goal`
- 如果没有参数，从当前学习上下文中获取精通度和复习时间数据

## 执行策略

1. 从学习上下文中提取精通度、FSRS 复习时间、错误记录等数据
2. 识别薄弱环节和需要复习的节点
3. 基于间隔重复原理制定复习时间线

## 输出要求

按如下格式输出学习计划：

### 学习计划：{概念/目标}

#### 当前状态
- 精通度：{描述性语言}
- 记忆稳定性：{描述性语言}
- 薄弱环节：{列举需要关注的点}

#### 复习安排

**今日**
- [ ] {复习任务 1}（预计 X 分钟）
- [ ] {复习任务 2}（预计 X 分钟）

**本周**
- [ ] {复习任务 3}
- [ ] {复习任务 4}

**下周**
- [ ] {复习任务 5}

#### 建议使用的技能
- `/error-revisit` — 针对 {薄弱概念}
- `/variation-practice` — 巩固 {核心概念}
- ...

确保：
- 任务可操作（不是"复习X"而是"用 /four-level 重新理解X的第3层"）
- 时间估算合理
- 优先级明确
