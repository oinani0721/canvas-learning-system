# 关系追问

深入探讨两个概念之间的关联，揭示隐含的联系和区别。

**用法**: `/relation-probe <概念A> 和 <概念B>` 或 `/relation-probe <连线上的两个节点>`

## 输入适配

用户提供的信息如下：

$ARGUMENTS

请将上述内容解析为：
- 提取两个要对比/关联的概念作为 `concept_a` 和 `concept_b`
- 如果只有一个概念，从学习上下文（相关节点部分）中找到已有连线的关联概念

## 输出要求

按如下格式输出关系分析：

### 关系追问：{概念A} ↔ {概念B}

#### 1. 表面关系
- 最显而易见的联系是什么

#### 2. 深层关系
- 底层原理上的联系
- 为什么它们经常一起出现

#### 3. 关键区别
- 容易混淆的地方
- 本质上的不同

#### 4. 追问链
一系列递进式问题，帮助深入理解两者的关系：
1. {追问 1}
2. {追问 2}
3. {追问 3}
4. ...（3-5 个追问）

#### 5. 一句话总结
用一句话概括两者的核心关系

确保：
- 追问链有逻辑递进
- 覆盖"为什么"和"什么时候"的维度
- 基于学生当前的精通度调整深度
