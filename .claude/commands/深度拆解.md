# 🎯 深度拆解Agent - 暴露似懂非懂概念的理解盲区

**这个Agent是干什么的？**

深度拆解Agent专门处理"似懂非懂"的概念（紫色节点）。它会基于你已有的理解，生成3-10个深度检验问题，帮助你发现理解中的盲区、误区和不够深入的地方，从"似懂非懂"过渡到"完全理解"（绿色）。

**核心能力**：检验真实理解，暴露知识盲区

**关键区别**：需要你已有的理解作为输入（vs 基础拆解不需要）

---

## 📝 什么时候用这个Agent？

### ✅ 适用场景

- **似懂非懂**：感觉懂了，但说不清楚
- **紫色节点**：评分60-79分的节点
- **有初步理解**：你已经填写了黄色理解节点
- **检验理解**：想知道自己到底懂不懂
- **发现盲区**：感觉理解有漏洞但不知道在哪

### ❌ 不适用场景

- **完全不懂**：如果你没有任何理解，应该用 `/基础拆解`
- **已经理解**：如果评分≥80分，不需要拆解了
- **需要记忆**：如果是记不住而不是理解问题，用 `/记忆锚点`

## ⚙️ Agent工作原理

### 输入要求

- **问题内容**：原始问题或概念
- **你的理解**：**必填** - 你对这个概念的当前理解
- **参考材料**：可选 - 原始材料用于对比

### 输出内容

生成3-10个深度检验问题，包含4种类型：

1. **对比型**：和XX概念有什么本质区别？
2. **原因型**：为什么要这样设计？
3. **应用型**：在XX场景下如何应用？
4. **边界型**：什么情况下会失效？

每个问题都针对你的理解盲区设计。

### 处理流程

```
输入: 问题 + 你的理解
    ↓
分析理解的准确性和完整性
    ↓
识别盲区（遗漏、误解、模糊）
    ↓
生成针对性的检验问题
    ↓
你重新思考和填写理解
    ↓
再次评分 → 目标80+分
```

## 🎓 实际案例

### 案例1：深度拆解"逆否命题"

**你的理解**（紫色节点，65分）：
```
逆否命题就是把原命题的条件和结论都否定，然后交换位置。
它和原命题是等价的，所以可以用来证明。
```

**Agent识别的盲区**：
- ✓ 知道定义
- ✗ 没说清楚"否定"的含义
- ✗ 不知道为什么等价
- ✗ 不知道什么时候用

**生成的深度问题**：
1. （对比型）逆否命题、逆命题、否命题三者有什么区别？哪个等价哪个不等价？
2. （原因型）为什么逆否命题和原命题等价？能用真值表证明吗？
3. （应用型）什么时候用逆否命题证明更简单？能举个实际例子吗？
4. （边界型）"否定"具体指什么？对于复杂命题如何正确否定？

**填写新理解后**：85分 → 变绿色 ✅

## 🆘 常见问题

**Q: 为什么必须提供我的理解？**
A: 深度拆解的核心是找出你理解中的盲区。没有你的理解，Agent无法知道你哪里有问题。

**Q: 我的理解很短可以吗？**
A: 可以，但理解越详细，Agent生成的问题越针对性。建议至少2-3句话。

**Q: 深度拆解后还是不懂怎么办？**
A: 说明需要补充解释而不是拆解。推荐使用 `/澄清路径` 或 `/口语化解释`。

**Q: 和基础拆解有什么区别？**
A:
- **基础拆解**：你完全不懂 → 降低门槛的引导性问题
- **深度拆解**：你似懂非懂 → 检验理解的深度问题

## 💡 使用技巧

### 技巧1：提供详细的理解

❌ 不好的输入：
```
"我知道是什么"
```

✅ 好的输入：
```
"逆否命题是把原命题的假设和结论都否定后交换位置。
它和原命题逻辑等价，所以可以用来证明原命题。
我不太确定为什么它们等价，还有什么时候应该用它。"
```

### 技巧2：明确说出不确定的地方

在理解中加上：
- "我不太确定..."
- "我不知道为什么..."
- "我不清楚什么时候..."

这样Agent会针对这些盲区生成问题。

### 技巧3：与评分配合

```
填写理解 → /评分 → 60-79分 → /深度拆解 → 优化理解 → /再次评分 → 80+分
```
