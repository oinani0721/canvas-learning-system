# 错题重访

基于学生的历史错误记录，生成针对性复习内容，帮助从错误中学习。

**用法**: `/error-revisit <概念名称>` 或 `/error-revisit`（自动从上下文获取错误记录）

## 输入适配

用户提供的信息如下：

$ARGUMENTS

请将上述内容解析为：
- 如果有具体概念名，作为 `concept`
- 如果没有参数，从当前学习上下文（system prompt 中的"历史错误"部分）获取错误记录

## 执行策略

1. 从学习上下文中提取该概念的历史错误记录
2. 分析每个错误的类型和根因
3. 针对每个错误生成纠正性解释和强化练习

## 输出要求

按如下格式输出错题复习：

### 错题复习：{概念名称}

**错误 1**: {错误描述}
- 错误原因：为什么会犯这个错
- 正确理解：应该怎么想
- 强化练习：一道帮助巩固正确理解的题目

**错误 2**: {错误描述}
- 错误原因：为什么会犯这个错
- 正确理解：应该怎么想
- 强化练习：一道帮助巩固正确理解的题目

...依此类推

### 防错清单
- [ ] 要点 1：避免 X 错误的关键检查
- [ ] 要点 2：...

确保：
- 语言积极正面（不是"你错了"而是"下次注意"）
- 每个错误都有对应的强化练习
- 防错清单可作为后续复习的检查项
