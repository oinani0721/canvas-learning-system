# 🎯 问题拆解Agent - 针对应用题生成突破性问题

**这个Agent是干什么的？**

问题拆解Agent专门处理应用题和问题求解场景。当你面对一道题目"不知道从哪里入手"时，它会生成突破性的引导问题，帮助你找到解题思路和突破口。

**核心能力**：从"不会做"到"找到思路"

**适用对象**：紫色节点的应用题场景

---

## 📝 什么时候用这个Agent？

### ✅ 适用场景

- **应用题求解**：面对一道题不知道怎么做
- **卡在某一步**：做题做到一半卡住了
- **不知道用什么**：知道概念但不知道怎么应用
- **紫色应用节点**：理论懂了但不会做题

### ❌ 不适用场景

- **概念不理解**：如果是概念不懂，应该用 `/基础拆解` 或 `/深度拆解`
- **需要完整解答**：如果需要看例题，应该用 `/例题教学`
- **记忆问题**：如果是记不住公式，用 `/记忆锚点`

## ⚙️ Agent工作原理

### 输入要求

- **题目内容**：必填 - 完整的题目描述
- **你的尝试**：可选 - 你已经尝试了什么
- **卡点**：可选 - 你卡在哪一步

### 输出内容

生成3-5个突破性问题，帮助你：

1. **识别类型**：这是什么类型的题目？
2. **找关键信息**：题目给了什么条件？要求什么？
3. **选择方法**：应该用什么概念/定理/方法？
4. **构造映射**：如何把题目条件映射到概念？
5. **分步骤**：第一步应该做什么？

### 处理流程

```
输入: 题目 + 你的尝试
    ↓
识别题目类型和考点
    ↓
分析你的尝试和卡点
    ↓
生成突破性引导问题
    ↓
你回答问题找到思路
    ↓
开始解题
```

## 📊 与其他拆解Agent的区别

| Agent | 适用场景 | 目标 | 输出特点 |
|-------|---------|------|---------|
| **基础拆解** | 概念完全不懂 | 建立基础认识 | 引导性问题（3-7个） |
| **深度拆解** | 概念似懂非懂 | 暴露理解盲区 | 检验性问题（3-10个） |
| **问题拆解** | **不会做题** | **找到解题思路** | **突破性问题（3-5个）** |

## 🎓 实际案例

### 案例1：鸽笼原理应用题

**题目**：
```
在任意367个人中，至少有多少人在同一天生日？
```

**你的状态**：知道鸽笼原理，但不知道怎么用

**Agent生成的突破性问题**：

1. **识别类型**：这道题考的是什么？能联想到鸽笼原理的什么特征？
   💡 提示：注意"至少"这个词

2. **构造鸽笼**：在这道题里，什么是"鸽笼"，什么是"东西"？
   💡 提示：一年有多少天？

3. **应用原理**：如果有n个鸽笼，n+1个东西，能得出什么结论？
   💡 提示：367和366的关系

4. **回答问题**：根据鸽笼原理，至少有几个人生日相同？
   💡 提示：366个"天"，367个"人"

**填写理解后**：
```
哦！我明白了：
- 鸽笼 = 一年的366天（闰年）
- 东西 = 367个人
- 367 > 366，所以至少有2个人在同一天生日
```

评分：82分 → 变绿色 ✅

## 🆘 常见问题

**Q: 问题拆解和基础拆解有什么区别？**
A:
- **基础拆解**：拆解概念（帮你理解"什么是鸽笼原理"）
- **问题拆解**：拆解题目（帮你"怎么用鸽笼原理解题"）

**Q: 问题拆解后我还是不会做怎么办？**
A: 说明你需要看完整例题。使用 `/例题教学` 查看详细解答步骤。

**Q: 可以用问题拆解来理解概念吗？**
A: 不行。问题拆解专注于解题思路，不适合概念理解。概念理解请用 `/基础拆解` 或 `/深度拆解`。

**Q: 我已经会做了，还需要问题拆解吗？**
A: 不需要。如果你已经理解了解题方法，直接填写解答然后 `/评分` 即可。

## 💡 使用技巧

### 技巧1：说明你的尝试

❌ 不好的输入：
```
"不会做"
```

✅ 好的输入：
```
"我知道要用鸽笼原理，但不知道怎么构造鸽笼。
我尝试把人当鸽笼，但感觉不对。"
```

### 技巧2：明确卡点

告诉Agent你卡在哪一步：
- "不知道从哪里入手"
- "不知道用什么方法"
- "知道方法但不知道怎么构造"
- "做到XX步骤卡住了"

### 技巧3：题目描述要完整

确保提供：
- ✓ 题目条件
- ✓ 题目要求
- ✓ 你的尝试（如果有）

## 🎯 典型应用场景

```

```

```
2. 应该从哪个运算符开始计算？
3. 计算顺序是什么？
```

### 场景3：应用题不知道用什么概念
```
题目：实际问题（如朋友数问题）
    ↓
/问题拆解 → 引导问题：
1. 这道题的本质是什么类型？
2. 题目条件暗示了什么概念？
3. 学过的哪个定理/原理可以用？
```
