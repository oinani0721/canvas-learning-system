# 🎯 生成检验问题Agent - Epic 4无纸化检验系统的核心引擎

**这个Agent是干什么的？**

生成检验问题Agent是Epic 4无纸化回顾检验系统的智能核心。它分析你在原白板上填写的黄色理解，为红色/紫色节点生成针对性的检验问题，用于创建检验白板。让你在不看资料的情况下复现知识，暴露理解盲区。

**核心能力**：智能生成检验问题，实现无纸化知识复现

**Epic 4地位**：检验白板自动生成的关键Agent

---

## 📝 什么时候用这个Agent？

### ✅ 适用场景

- **生成检验白板**：自动为红/紫节点生成检验问题
- **无纸化复习**：不看原白板，从头复现知识
- **暴露盲区**：通过检验问题发现理解漏洞
- **UltraThink v1**：智能策略系统自动调用生成检验问题
- **知识验证**：检验自己是否真正理解

### ❌ 不适用场景

- **学习阶段**：如果还在学习，不要用检验问题（先学完）
- **绿色节点**：已经理解的节点（≥80分）不需要检验
- **手动检验**：如果想手动写检验问题，不需要这个Agent

## ⚙️ Agent工作原理

### 输入要求

- **节点内容**：红色/紫色节点的问题内容
- **黄色理解**：**关键输入** - 你对这个节点的理解
- **节点颜色**：红色 or 紫色（决定问题类型）

### 生成策略：根据颜色和理解程度

#### 🔴 红色节点（完全不懂）→ 1-2个问题

**问题类型**：
1. **突破型问题**：换个角度帮助理解
2. **基础型问题**：降低门槛的简单问题

**示例**：
```
原节点：什么是逆否命题？
黄色理解：完全不懂

生成检验问题：
1. （突破型）生活中有没有"反过来说意思不变"的例子？
   提示：想想"有A就有B"可以怎么换个说法

2. （基础型）如果原话是"下雨→地湿"，
   你能写出它的逆否命题吗？
```

#### 🟣 紫色节点（似懂非懂）→ 2-3个问题

**问题类型**：
1. **检验型问题**：测试是否真正理解
2. **应用型问题**：能否迁移到新场景

**示例**：
```
原节点：如何应用逆否命题？
黄色理解：知道定义，但不知道什么时候用

生成检验问题：
1. （检验型）为什么逆否命题和原命题等价？
   能用真值表或逻辑推理证明吗？

2. （应用型）什么时候用逆否命题证明更简单？
   能举一个实际例子吗？

3. （区分型）逆命题、否命题、逆否命题，
   哪个和原命题等价？为什么？
```

### UltraThink智能策略增强

在UltraThink v1生成时，Agent会：
1. 分析黄色理解的程度（0-100%）
2. 根据理解程度决定问题难度
3. 理解度低（0-30%）→ 生成基础引导问题
4. 理解度中（40-70%）→ 生成深度检验问题
5. 理解度高（80%+）→ 生成应用拓展问题

## 🆘 常见问题

**Q: 生成检验问题Agent和深度拆解Agent有什么区别？**
A:
- **深度拆解Agent**：在原白板上使用，拆解概念帮助理解
- **生成检验问题Agent**：为检验白板生成问题，验证理解

**Q: 为什么生成的问题有时很简单，有时很难？**
A: 因为Agent分析了你的黄色理解：
- 理解程度低（0-30%）→ 简单基础问题
- 理解程度中（40-70%）→ 深度检验问题
- 理解程度高（80%+）→ 应用拓展问题

**Q: 检验白板生成后可以修改问题吗？**
A: 可以！生成的问题是建议：
- 可以手动调整问题难度
- 可以添加你自己的问题
- 可以删除不需要的问题

**Q: 检验白板和原白板有什么区别？**
A:
- **原白板**：学习白板，有AI辅助，边学边填
- **检验白板**：检验白板，初始只有问题，测试复现能力

**但检验白板也支持所有Agent操作**！卡住时可以调用Agent。

## 💡 生成高质量检验问题的技巧

### 技巧1：黄色理解要详细

❌ 不好的黄色理解：
```
"不懂"
```

✅ 好的黄色理解：
```
"我知道逆否命题是把条件和结论否定后交换，
但不理解为什么它和原命题等价，
也不知道什么时候应该用逆否命题证明。"
```

**原因**：理解越详细，生成的问题越针对你的盲区。

### 技巧2：写综述提供宏观信息

在UltraThink v1生成时：
```
--overview "我学习了逆否命题、逆命题、否命题，
理解了逆否命题的定义，但容易和其他两个混淆，
还不知道实际应用场景。"
```

系统会根据综述：
- 识别易混淆点 → 生成区分性问题
- 识别遗漏点 → 生成补充问题
- 调整问题侧重点

### 技巧3：迭代优化检验问题

```
第一次生成检验白板
    ↓
填写后发现问题太简单/太难
    ↓
调整黄色理解的详细程度
    ↓
重新生成或手动调整问题
```

## 🎯 Epic 4无纸化检验系统

### 完整工作流

```
Phase 1: 原白板学习
├─ 学习材料
├─ 拆解问题（/基础拆解，/深度拆解）
├─ 补充解释（/澄清路径，/口语化解释）
├─ 填写黄色理解
└─ 评分 → 识别红色/紫色节点

Phase 2: 生成检验白板（Agent自动）
├─ 调用 verification-question-agent
├─ 提取红色/紫色节点
├─ 分析黄色理解
├─ 生成检验问题
├─ 创建检验白板.canvas
└─ 输出：检验白板文件

Phase 3: 无纸化检验
├─ 打开检验白板（不看原白板！）
├─ 阅读检验问题
├─ 填写黄色理解
└─ 发现哪些真的懂，哪些其实不懂

Phase 4: 检验白板上动态学习
├─ /评分 黄色节点
├─ <60分 → /澄清路径 或 /口语化解释
├─ 60-79分 → /深度拆解
├─ /记忆锚点 巩固记忆
├─ 持续迭代
└─ 直到80%节点≥80分

Phase 5: 生成v2检验白板（可选）
├─ 只包含未通过的节点（<80分）
├─ 重复Phase 3-4
└─ 直到所有节点通过
```

### 何时停止迭代？

满足以下至少3个条件：
- ✅ 绿色占比 ≥ 80%
- ✅ 检验白板节点数接近原白板的50-70%
- ✅ 至少生成3个解释文档
- ✅ 至少添加2个原创节点
- ✅ 无红色节点

## 🎓 最佳实践

### ✅ 检验白板的价值

1. **暴露盲区**：不看资料复现，发现真正的盲区
2. **主动回忆**：主动回忆比被动阅读效果好10倍
3. **动态学习**：检验白板支持所有Agent，可以持续学习
4. **版本化迭代**：v1→v2→v3，逐步收敛到完全理解
5. **知识复现**：最终能从头复现整个知识体系

### ✅ 使用检验白板的技巧

1. **不要偷看原白板**：这是最重要的原则
2. **卡住时调用Agent**：检验白板也可以用所有Agent
3. **对比原白板找差距**：完成后对比，发现遗漏
4. **持续扩展**：不断添加节点，直到接近原白板

### ❌ 常见错误

- ❌ 边看原白板边填检验白板（失去检验意义）
- ❌ 检验白板生成后就不管了（应该持续学习）
- ❌ 只填一次就评分（应该迭代优化）
- ❌ 不用Agent辅助（检验白板支持所有Agent！）
