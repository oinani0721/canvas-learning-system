# 检验问题生成

基于学生的理解盲点，生成有针对性的深度检验问题，帮助发现理解差距。

**用法**: `/verify-question <概念或材料内容> [我的理解: ...]`

请先读取 Agent 定义获取完整指导：

```
Read .claude/agents/verification-question-agent.md
```

读取后，严格按照定义中的 System Prompt 执行任务。

## 输入适配

用户提供的内容如下：

$ARGUMENTS

请解析上述内容：
- 主体内容作为节点的 `content`
- 如果用户表示"完全不理解"，视为 `type: "red"`（红色节点），生成1-2个突破型/基础型问题
- 如果用户表示"似懂非懂"或附带了自己的理解，视为 `type: "purple"`（紫色节点），生成2-3个检验型/应用型问题
- 如果用户包含"我的理解是..."等表述，提取为 `related_yellow` 内容
- 默认视为紫色节点（似懂非懂）

## 输出适配

**不要输出 JSON**。以 Markdown 格式输出检验问题：

### 检验问题

对每个问题按如下格式输出：

**问题 1** [类型：突破型/检验型/应用型/综合型 | 难度：基础/深度]
问题文本内容
💡 提示（如有）
> 生成理由：为什么生成这个问题的解释

**问题 2** [类型：... | 难度：...]
...

确保问题具有针对性，能揭示理解盲点而非仅测试记忆。
