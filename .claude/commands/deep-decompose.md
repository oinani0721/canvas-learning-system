# 深度拆解

针对"似懂非懂"的概念，生成3-10个深层次检验型问题，暴露理解盲点，帮助从"似懂非懂"过渡到"完全理解"。

**用法**: `/deep-decompose <材料内容> [我的理解: ...]`

建议附带你的个人理解，以便生成更有针对性的检验问题。

请先读取 Agent 定义获取完整指导：

```
Read .claude/agents/deep-decomposition.md
```

读取后，严格按照定义中的 System Prompt 和深度拆解策略执行任务。

## 输入适配

用户提供的材料/概念如下：

$ARGUMENTS

请将上述内容解析为：
- 材料内容作为 `material_content`
- 从内容中自动识别 `topic`
- 如果用户包含"我的理解是..."、"我理解为..."、"我觉得..."等表述，提取为 `user_understanding`
- 如果没有明确的用户理解，提示用户最好提供自己的理解以获得更有针对性的问题

## 输出适配

**不要输出 JSON**。以 Markdown 格式输出深度拆解结果：

### 深度检验问题

对每个问题按如下格式输出：

**问题 1** [类型：对比型 | 难度：深度]
问题文本内容
💡 提示：具体引导提示

**问题 2** [类型：原因型 | 难度：深度]
问题文本内容
💡 提示：具体引导提示

...依此类推（共3-10个问题）

确保：
- 问题数量3-10个，难度均为"深度"
- 问题类型包含：对比型、原因型、应用型、边界型（至少3种）
- 问题能够测试真实理解，暴露"似懂非懂"的盲点
