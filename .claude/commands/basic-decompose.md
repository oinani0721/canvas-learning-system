# 基础拆解

将"看不懂"的难懂材料拆解为3-7个简单、引导性的子问题，帮助从"完全不理解"逐步过渡到"有初步理解"。

**用法**: `/basic-decompose <难懂的材料内容或概念>`

请先读取 Agent 定义获取完整指导：

```
Read .claude/agents/basic-decomposition.md
```

读取后，严格按照定义中的 System Prompt 和拆解策略执行任务。

## 输入适配

用户提供的材料/概念如下：

$ARGUMENTS

请将上述内容解析为：
- 较长的文本作为 `material_content`（要拆解的难懂材料）
- 从内容中自动识别 `topic`（主题名称）
- 如果用户附带了自己的初步理解，提取为 `user_understanding`
- 否则 `user_understanding` 为 null

## 输出适配

**不要输出 JSON**。以 Markdown 格式输出拆解结果：

### 拆解结果

对每个子问题按如下格式输出：

**问题 1** [类型：定义型]
问题文本内容
💡 提示：引导性提示（如有）

**问题 2** [类型：实例型]
问题文本内容

...依此类推（共3-7个问题）

确保：
- 问题数量3-7个，难度均为"基础"
- 问题类型包含：定义型、实例型、对比型、探索型
- 问题之间有逻辑递进关系
- 语言简单口语化
