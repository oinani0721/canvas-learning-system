# 理解评分

使用4维度评分标准（准确性、形象性、完整性、原创性）评估用户对概念的理解质量。

**用法**: `/score <问题/概念> --- <你的理解>`

用 `---` 分隔问题和你的理解，例如：
- `/score 什么是逆否命题 --- 逆否命题就是把原命题反过来说，而且意思相同`
- `/score 傅里叶变换的本质 --- 我觉得就是把复杂的波拆成简单的正弦波`

请先读取 Agent 定义获取完整指导：

```
Read .claude/agents/scoring-agent.md
```

读取后，严格按照定义中的4维度评分标准和"词可达意"哲学执行评估。

## 输入适配

用户提供的内容如下：

$ARGUMENTS

请解析上述内容：
- 用 `---`、`|`、`>>>` 或换行符作为分隔符
- 分隔符前面的内容作为 `question_text`（问题/概念）
- 分隔符后面的内容作为 `user_understanding`（用户的理解）
- 如果没有找到分隔符，提示用户需要同时提供"问题"和"你的理解"
- `reference_material` 可根据问题内容推断

## 输出适配

**不要输出 JSON**。以 Markdown 格式输出评分报告：

### 评分结果

**总分**: XX/100 分 [通过/似懂非懂/未通过]

| 维度 | 分数 | 说明 |
|------|------|------|
| 准确性 | XX/25 | 简要评价 |
| 形象性 | XX/25 | 简要评价 |
| 完整性 | XX/25 | 简要评价 |
| 原创性 | XX/25 | 简要评价 |

### 反馈建议

具体的改进建议（100-200字）
