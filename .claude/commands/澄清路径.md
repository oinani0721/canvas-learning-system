# 🎯 澄清路径Agent - 系统化深度澄清文档（1500+词）

**这个Agent是干什么的？**

澄清路径Agent是最深度、最系统化的解释工具。它会生成1500+词的完整澄清文档，采用4步澄清法：问题澄清 → 概念拆解 → 深度解释 → 验证总结。适合彻底搞懂一个复杂概念。

**核心能力**：深度系统化澄清，从根本上解决理解问题

**输出格式**：生成 `.md` 文件（1500+词，4个section）

**定位**：UltraThink智能策略中的"补充解释"首选方案

---

## 📝 什么时候用这个Agent？

### ✅ 适用场景

- **复杂抽象概念**：概念很复杂，需要深度系统化讲解
- **理解度极低**：UltraThink分析显示理解度0-20%
- **反复看不懂**：看了多次教材还是不理解
- **需要彻底搞懂**：不满足于浅层理解，要完全弄明白
- **补充解释策略**：智能v1生成时推荐的"澄清路径"节点

### ❌ 不适用场景

- **快速理解**：如果只需要简单讲解，用 `/口语化解释`
- **对比概念**：如果要对比多个概念，用 `/对比表`
- **记忆问题**：如果理解了但记不住，用 `/记忆锚点`
- **分层次学习**：如果需要渐进式理解，用 `/四层次解释`

## ⚙️ Agent工作原理

### 输入要求

- **概念名称**：必填 - 需要澄清的概念
- **困惑点**：可选但强烈推荐 - 你具体哪里不懂

### 输出内容：4步澄清法（1500+词）

#### 📌 第1步：问题澄清（300-400词）
```
- 你的困惑到底是什么？
- 为什么这个概念难理解？
- 需要澄清的核心问题是什么？
- 从什么角度切入最容易理解？
```

#### 📌 第2步：概念拆解（400-500词）
```
- 把复杂概念拆解为基础组件
- 每个组件单独解释
- 建立概念之间的关系
- 从简单到复杂递进
```

#### 📌 第3步：深度解释（500-600词）
```
- 概念的本质是什么？
- 为什么要这样定义？
- 背后的原理是什么？
- 与其他概念的关系？
- 实际应用场景
- 生动例子和类比
```

#### 📌 第4步：验证总结（200-300词）
```
- 核心要点总结（3-5个）
- 自我检验问题（3个）
- 常见误区提醒
- 下一步学习建议
```

### 生成的文件

- **文件名**：`{概念}-澄清路径-{时间戳}.md`
- **位置**：Canvas同目录
- **长度**：1500+词（最详细的解释Agent）

## 🆘 常见问题

**Q: 澄清路径和口语化解释有什么区别？**
A:
- **口语化解释**：像聊天，轻松易懂，800-1200词
- **澄清路径**：系统文档，深度澄清，1500+词，4步结构

选择标准：
- 快速理解 → 口语化解释
- 彻底搞懂 → 澄清路径

**Q: 1500+词太长了，一定要全读完吗？**
A: 建议分步阅读：
1. 第一次：读第1步（问题澄清）和第4步（总结）
2. 第二次：仔细读第2步（概念拆解）
3. 第三次：深入读第3步（深度解释）
4. 最后：用第4步的检验问题测试自己

**Q: 澄清路径生成后还是不懂怎么办？**
A:
1. 尝试 `/四层次解释` （从新手层开始）
2. 尝试 `/对比表` （如果是概念混淆）
3. 尝试 `/例题教学` （通过例题理解）
4. 考虑是否有前置概念不懂

**Q: 什么时候系统会自动推荐澄清路径？**
A: UltraThink智能v1生成时，如果检测到：
- 理解度0-20% →自动生成"补充解释"策略 → 推荐澄清路径
- 或者你可以手动强制使用本命令
