# 问题拆解

针对"似懂非懂"的概念，生成2-5个少而精的检验型和应用型问题，帮助发现理解盲区。

**用法**: `/question-decompose <材料内容> [我的理解: ...]`

建议附带你的个人理解。

请先读取 Agent 定义获取完整指导：

```
Read .claude/agents/question-decomposition.md
```

读取后，严格按照定义中的 System Prompt 和问题拆解策略执行任务。

## 输入适配

用户提供的材料/概念如下：

$ARGUMENTS

请将上述内容解析为：
- 材料内容作为 `material_content`
- 从内容中自动识别 `topic`
- 如果用户包含"我的理解是..."、"我理解为..."、"我觉得..."等表述，提取为 `user_understanding`
- 如果没有用户理解，提示用户最好提供自己的理解

## 输出适配

**不要输出 JSON**。以 Markdown 格式输出：

### 检验问题

对每个问题按如下格式输出：

**问题 1** [类型：检验型]
问题文本内容
💡 提示：引导性提示

**问题 2** [类型：应用型]
问题文本内容
💡 提示：引导性提示

...依此类推（共2-5个问题，少而精）

确保：
- 问题数量2-5个，少而精
- 问题类型至少2种：检验型、应用型、对比型、推理型
- 优先使用检验型和应用型
- 问题针对用户理解中的具体盲区
