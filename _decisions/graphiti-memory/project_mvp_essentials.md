# ⛔⛔⛔ 10 条开发纪律（DD-01 ~ DD-10）

> **用户2026-03-16反复强调，多个session仍被无视。违反任何一条 = 功能性错误。**
> **详细规则见 `.claude/rules/development-discipline.md`**

| # | 规则 | 核心要求 |
|---|------|---------|
| DD-01 | 学术实证 | 功能必须有论文/案例支撑。⛔ Context7+WebSearch 实际查证 |
| DD-02 | 符合实际 | 不脱离用户初衷、实际代码、Obsidian 环境。⛔ search_memory_facts |
| DD-03 | 禁 mock | 禁止 mock API/模拟数据/TODO空函数。编辑后 LSP/ruff |
| DD-04 | 参考案例落地 | ⛔ Context7+WebSearch 查参考实现。禁止东拼西凑 |
| DD-05 | Pencil 范式 | 前后端先 Pencil 创建 UI 范式展示给用户确认 |
| DD-06 | Obsidian 适配 | CSS隔离/Plugin API/主题兼容/Electron |
| DD-07 | 对抗性测试 | 真实数据严苛测试，验证符合用户预期 |
| DD-08 | Graphiti 初衷 | 积极搜索用户初衷和决策，功能开发以用户初衷为首位 |
| DD-09 | 增量提问 | 不确定就问用户，可用 Pencil 展示场景 |
| DD-10 | 防功能蔓延 | 新功能对照上方 MVP 刚需清单，不在清单中的不做 |
